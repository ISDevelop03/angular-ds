# API Reference

Property Type Default Description
Component Inputs
content TemplateRef<any> | string - Required. The content to display in the tooltip. Can be either:
  • String: Simple text content rendered directly. Use for short, simple tooltip messages.
  • TemplateRef: Angular template reference for complex content. Use for rich HTML content, multiple lines, formatted text, or interactive elements. The template is rendered using ngTemplateOutlet.
The tooltip is only rendered if content is provided (conditional rendering with *ngIf).
position 'top' | 'topRight' | 'topLeft' | 'bottom' | 'bottomRight' | 'bottomLeft' | 'left' | 'right' 'top' The position of the tooltip relative to the trigger element. Available options:
  • 'top': Above the element, centered
  • 'topRight': Above the element, aligned to the right
  • 'topLeft': Above the element, aligned to the left
  • 'bottom': Below the element, centered
  • 'bottomRight': Below the element, aligned to the right
  • 'bottomLeft': Below the element, aligned to the left
  • 'left': To the left of the element, vertically centered
  • 'right': To the right of the element, vertically centered
Each position has corresponding arrow styling that points to the trigger element.
disabled boolean false When true, disables the tooltip, preventing it from showing on hover or focus. Disabled tooltips won't respond to mouse enter/leave or focus events. Use for tooltips that should be temporarily unavailable or conditionally shown.
delay number 200 The delay in milliseconds before showing the tooltip after the user hovers over or focuses the trigger element. This prevents tooltips from appearing too quickly on accidental hovers. The delay is applied using setTimeout and is cleared if the user moves away before the delay completes.
className string '' Optional additional CSS classes to apply to the tooltip container. Useful for custom styling, spacing, or layout adjustments. The component has default styling with dark background, white text, rounded corners, shadow, and positioning based on the position property.
theme any defaultTheme Optional custom theme object to override the default styling. If not provided, the component uses the default theme. The theme object should follow the same structure as defaultTheme, including base, positions, and arrow properties. Use for custom color schemes, sizes, or styling variations.
style Record<string, string> {} Optional inline styles object to apply directly to the tooltip element. Each key-value pair in the object is applied as a CSS property-value pair. Useful for dynamic styling, custom colors, or specific positioning adjustments that can't be achieved with CSS classes alone. Styles are applied after the tooltip becomes visible.

Additional Notes:

  • Content Projection: The tooltip component uses content projection (ng-content) to wrap the trigger element. Any content placed between the <ds-tooltip> tags will act as the trigger element. The tooltip appears relative to this trigger.
  • Trigger Events: The tooltip appears on:
    • Mouse enter: When the user hovers over the trigger element
    • Focus in: When the trigger element receives focus (keyboard navigation)
    The tooltip hides on:
    • Mouse leave: When the user moves the mouse away
    • Focus out: When the trigger element loses focus
  • Delay Behavior: The delay property controls how long the user must hover or focus before the tooltip appears. If the user moves away before the delay completes, the tooltip won't show. This prevents tooltips from appearing on accidental hovers or quick mouse movements.
  • Positioning: The tooltip is positioned absolutely relative to the trigger element:
    • Top/Bottom positions: Tooltip appears above or below, with optional left/right alignment
    • Left/Right positions: Tooltip appears to the side, vertically centered
    • Arrow: Each position has a corresponding arrow that points to the trigger element, created using CSS borders
    The tooltip has a maximum width of 300px (max-w-[300px]) to prevent it from becoming too wide.
  • Animations: The tooltip includes smooth show/hide animations:
    • Show: Fade in with scale up (150ms ease-out)
    • Hide: Fade out with scale down (100ms ease-in)
    The animations use Angular's animation system with opacity and transform transitions.
  • Styling: The default tooltip styling includes:
    • Background: Dark gray (bg-neutral-900), with dark mode support (dark:bg-neutral-800)
    • Text: White (text-white), small font size (text-sm), medium weight (font-medium)
    • Padding: px-3 py-2 (12px horizontal, 8px vertical)
    • Border radius: rounded-lg
    • Shadow: shadow-tooltip
    • Z-index: z-[9999] to ensure it appears above other content
    • Pointer events: pointer-events-none to prevent interaction with the tooltip itself
  • String vs Template Content: The component automatically detects whether content is a string or TemplateRef:
    • String: Rendered directly using interpolation ({{ content }})
    • TemplateRef: Rendered using ngTemplateOutlet, allowing for complex HTML structures
    Use strings for simple text, and TemplateRef for rich content with formatting, multiple elements, or interactive components.
  • Accessibility: The tooltip includes accessibility features:
    • role="tooltip" attribute for screen readers
    • aria-hidden attribute that toggles based on visibility state
    • Keyboard support via focus events
    Ensure the trigger element is keyboard accessible (e.g., has tabindex or is a focusable element like a button or link).
  • Use Cases: This component is ideal for:
    • Providing additional information or context
    • Explaining form field requirements or validation rules
    • Showing help text or tips
    • Displaying truncated content on hover
    • Providing contextual information for icons or buttons
  • Performance: The component uses efficient change detection:
    • Only renders the tooltip DOM when content is provided
    • Uses ChangeDetectorRef.detectChanges() to manually trigger change detection when showing/hiding
    • Cleans up timeouts on destroy to prevent memory leaks
  • Custom Styling: You can customize the tooltip appearance in multiple ways:
    • className: Add custom CSS classes for styling adjustments
    • style: Apply inline styles for dynamic or specific styling
    • theme: Provide a complete custom theme object for comprehensive styling changes
  • Arrow Positioning: The arrow automatically positions itself based on the position property:
    • Top positions: Arrow at the bottom, pointing down
    • Bottom positions: Arrow at the top, pointing up
    • Left position: Arrow on the right, pointing right
    • Right position: Arrow on the left, pointing left
    The arrow is created using CSS borders to form a triangle shape.
  • Multiple Tooltips: You can use multiple tooltips on the same page. Each tooltip manages its own visibility state and timing independently. The high z-index ensures tooltips appear above other content.
  • Responsive Behavior: The tooltip adapts to its content:
    • Width: Automatically adjusts to content width, with a maximum of 300px
    • Height: Automatically adjusts to content height
    • Text wrapping: Long text automatically wraps within the tooltip
