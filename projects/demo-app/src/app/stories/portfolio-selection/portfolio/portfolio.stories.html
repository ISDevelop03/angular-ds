Portfolio

# API Reference

Property Type Default Description
Component Inputs
portfolios Portfolio[] [] Required. Array of portfolio items to display in the dropdown. Each portfolio item is displayed as a selectable card. The portfolios are paginated based on perPage. See the Portfolio Interface section below for details.
selected Portfolio | null null The currently selected portfolio. Displayed in the main selector card at the top. When provided, this portfolio is shown as selected. When null, the first portfolio from portfolios array is used as the default selection.
holding Portfolio | null null Optional special portfolio item displayed at the top of the dropdown list (above the regular portfolios). Typically used for a "holding" or special portfolio option. When provided, it's displayed using the ds-portfolio-vision-card component. When null, it's not displayed.
show boolean false Initial visibility state of the dropdown. When true, the dropdown is open on component initialization. When false, the dropdown is closed. The dropdown can be toggled by clicking the selector card. When the dropdown is open, body scroll is disabled.
showDots boolean true When true, displays pagination dots and navigation arrows when there are multiple pages (totalPages > 1). When false, pagination controls are hidden. The dots indicate the current page and allow navigation between pages.
perPage number 4 Number of portfolio items to display per page. The portfolios array is paginated based on this value. The component automatically calculates totalPages based on portfolios.length and perPage. Use to control how many items are visible at once.
currentPage number 1 The currently active page number (1-based). Controls which set of portfolios is displayed. When changed, the component displays the portfolios for that page. The component automatically resets to page 1 if currentPage exceeds totalPages.
totalPages number 1 Total number of pages calculated from portfolios.length and perPage. The component automatically calculates this value using Math.ceil(portfolios.length / perPage). Used to determine pagination controls visibility and navigation limits.
totalItems number 0 Total number of items (for display purposes). Currently available as an input property for future use or custom implementations. The component primarily uses portfolios.length for calculations.
className string '' Optional additional CSS classes to apply to the portfolio component container. Useful for custom styling, layout adjustments, or responsive sizing (e.g., max-w-[250px]). The component has default styling with relative positioning, z-index, flex layout, and gap spacing.
Component Outputs (Events)
clicked EventEmitter<Portfolio> - Emitted when a portfolio item is selected (clicked). Provides the selected Portfolio object. This event is triggered when:
  • A regular portfolio item card is clicked
  • The holding portfolio (if provided) is clicked
When a portfolio is selected, the dropdown automatically closes and the selected portfolio is displayed in the selector card. Use to handle portfolio selection, update state, or trigger actions.
pageChange EventEmitter<number> - Emitted when the user navigates to a different page. Provides the new page number (1-based). Triggered when:
  • A pagination dot is clicked
  • The previous arrow button is clicked
  • The next arrow button is clicked
Use to sync the page state with your application or perform actions when the page changes.

# Portfolio Interface

The Portfolio interface defines the structure for portfolio items passed to the portfolios, selected, and holding input properties.

Property Type Default Description
id any - Required. Unique identifier for the portfolio item. Used to determine which portfolio is selected (compared with selectedPortfolio.id). Can be a string, number, or any value that can be used for comparison.
title string - Required. The title text displayed for the portfolio. Shown in the selector card and in each portfolio item card. This is the primary label for the portfolio (e.g., portfolio name, account name).
image string - Required. URL or path to the portfolio image. The image is displayed in the selector card and in each portfolio item card. Used for visual identification of the portfolio.
reference string undefined Optional reference text displayed in the selector card. Shown below or alongside the title. Use to provide additional information about the portfolio (e.g., account number, reference code, identifier).
