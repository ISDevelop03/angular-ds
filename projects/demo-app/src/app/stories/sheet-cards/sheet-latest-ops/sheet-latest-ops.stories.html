# API Reference

Property Type Default Description
Component Inputs
isDiscrete boolean false When true, enables discrete mode for the balance formatter. In discrete mode, the price value may be displayed in a more subtle or masked way (e.g., showing asterisks or reduced visibility). This is useful for privacy-sensitive applications where financial information should be partially hidden. The value is passed to the internal ds-balance-formatter component.
title string '' The title text displayed at the top-left of the card. Shown in bold text with a font size of 0.875rem (14px) and line height of 1rem (16px). The title appears on the same row as the price, aligned to the left. Use to describe the operation or transaction (e.g., "Virement", "Prélèvement", "Dépôt").
currency string - Required. The currency code or symbol for the price (e.g., "EUR", "USD", "MAD"). This is passed to the internal ds-balance-formatter component, which displays the currency alongside the formatted price value. The currency is typically shown as a prefix or suffix to the price.
price number - Required. The numeric price or amount value to display. This value is formatted and displayed using the ds-balance-formatter component, which handles number formatting, currency display, and discrete mode. The price is shown on the top-right of the card, aligned to the right, in bold text. If the price is falsy (null, undefined, 0, etc.), the balance formatter is not rendered.
dateOperation string - Required. The date of the operation (transaction date) displayed in the bottom-right section of the card. This is the date when the transaction was executed or processed. The date is displayed in bold text below the "Date opération" label. The value is also used as a condition for the ds-display-value component to determine if the date should be shown.
dateValue string - Required. The value date (date de valeur) displayed in the bottom-left section of the card. This is the date when the transaction value is effective or when funds are available. The date is displayed in bold text below the "Date valeur" label. The value is conditionally rendered using the ds-display-value component based on the dateOperation value.
className string '' Optional additional CSS classes to apply to the card container. Useful for custom styling, borders, backgrounds, spacing, or layout adjustments. The component has default styling with background (bg-level-3), padding (py-3 px-3.5), rounded corners (rounded), and flex layout with gap spacing.

Additional Notes:

  • Card Layout: The component displays a card with a two-section layout:
    • Top section: Title on the left and formatted price on the right
    • Bottom section: Two date fields side by side - "Date valeur" on the left and "Date opération" on the right
  • Balance Formatter: The component uses the ds-balance-formatter component internally to format and display the price. The formatter handles:
    • Number formatting with proper decimal places
    • Currency display (prefix or suffix)
    • Discrete mode for privacy (when isDiscrete: true)
    The balance formatter is only rendered if price is truthy.
  • Display Value Component: The component uses ds-display-value to conditionally render the price and dates. This component likely handles null/undefined checks and provides consistent display logic. The dates are only shown if dateOperation is truthy.
  • Date Display: The card displays two types of dates:
    • Date valeur (Value Date): The date when the transaction value is effective or when funds are available. This is important for accounting and banking purposes.
    • Date opération (Operation Date): The date when the transaction was executed or processed. This is the actual transaction date.
    Both dates are displayed in the same format (bold text, 0.875rem font size) with gray labels above them.
  • Discrete Mode: When isDiscrete is true, the price may be displayed in a masked or partially hidden format. This is useful for:
    • Privacy-sensitive applications
    • Screenshots or demos where sensitive data should be hidden
    • Compliance with data protection requirements
    The discrete mode is passed to the balance formatter, which handles the actual masking logic.
  • Styling: The card has consistent styling with:
    • Background: bg-level-3 (theme-based background color)
    • Padding: py-3 px-3.5 (vertical: 12px, horizontal: 14px)
    • Rounded corners: rounded
    • Gap spacing: gap-2 (8px) between sections
    • Font sizes: 0.875rem (14px) for most text, with bold styling for values
  • Responsive Design: The card uses flexbox for layout, making it responsive. The top section uses justify-between to space the title and price, while the bottom section displays dates side by side. The layout adapts to different container widths.
  • Use Cases: This component is designed for displaying financial transaction or operation information, such as:
    • Bank account transactions
    • Payment history
    • Operation logs
    • Financial statements
  • Required Fields: The following fields are required for the component to display properly:
    • currency - for currency display
    • price - for the amount (though it's conditionally rendered)
    • dateOperation - for the operation date
    • dateValue - for the value date
    The title is optional but recommended for clarity.
