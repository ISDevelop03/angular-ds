# API Reference

Property Type Default Description
Component Inputs
isDiscrete boolean false When true, enables discrete mode for the balance formatter. In discrete mode, the price value may be displayed in a more subtle or masked way (e.g., showing asterisks or reduced visibility). This is useful for privacy-sensitive applications where financial information should be partially hidden. The value is passed to the internal ds-balance-formatter component.
title string '' The title text displayed at the top-left of the card. Shown in bold text with a font size of 0.875rem (14px) and line height of 1rem (16px). The title appears on the same row as the price, aligned to the left. Use to describe the operation or transaction (e.g., "Virement", "Prélèvement", "Dépôt").
currency string - Required. The currency code or symbol for the price (e.g., "EUR", "USD", "MAD"). This is passed to the internal ds-balance-formatter component, which displays the currency alongside the formatted price value. The currency is typically shown as a prefix or suffix to the price.
price number - Required. The numeric price or amount value to display. This value is formatted and displayed using the ds-balance-formatter component, which handles number formatting, currency display, and discrete mode. The price is shown on the top-right of the card, aligned to the right, in bold text. If the price is falsy (null, undefined, 0, etc.), the balance formatter is not rendered.
dateOperation string - Required. The date of the operation (transaction date) displayed in the bottom-right section of the card. This is the date when the transaction was executed or processed. The date is displayed in bold text below the "Date opération" label. The value is also used as a condition for the ds-display-value component to determine if the date should be shown.
dateValue string - Required. The value date (date de valeur) displayed in the bottom-left section of the card. This is the date when the transaction value is effective or when funds are available. The date is displayed in bold text below the "Date valeur" label. The value is conditionally rendered using the ds-display-value component based on the dateOperation value.
className string '' Optional additional CSS classes to apply to the card container. Useful for custom styling, borders, backgrounds, spacing, or layout adjustments. The component has default styling with background (bg-level-3), padding (py-3 px-3.5), rounded corners (rounded), and flex layout with gap spacing.
