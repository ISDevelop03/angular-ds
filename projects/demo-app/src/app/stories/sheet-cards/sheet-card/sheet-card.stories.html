# API Reference

Property Type Default Description
Component Inputs
image string undefined Optional URL path to an image displayed on the left side of the card. The image is displayed in a 40px × 40px container with object-fit cover. By default, the image is rounded (circular), but this can be changed with removeImageRounded. If both image and icon are provided, the image takes precedence. The image uses the label as its alt text for accessibility.
icon string undefined Optional icon identifier displayed on the left side of the card when image is not provided. Should match an available icon in your icon library (e.g., 'home', 'settings', 'user'). The icon is displayed in a 40px × 40px circular container with a white background, border, and red color. The icon size is 20px.
label string undefined Optional label text displayed at the top of the card content area. The label styling depends on removeImageRounded: if true, it's bold and dark; if false, it's normal weight and gray. The label is also used as the alt text for the image when provided.
isDraggable string undefined When set to 'true' (as a string), enables draggable functionality. When enabled, a drag handle icon appears on hover at the left side of the card. The card also gets hover effects with padding adjustments (increased left padding, decreased right padding) to accommodate the drag handle. The drag handle is positioned absolutely and only visible on hover.
leftContent string | TemplateRef<any> undefined Optional custom content to display on the left side of the card, replacing the default image or icon. Can be either a string or an Angular TemplateRef. When a TemplateRef is provided, it's rendered using ngTemplateOutlet, allowing for complex custom layouts (e.g., multiple icons, custom graphics, or interactive elements). This takes precedence over both image and icon.
data any undefined Optional data object passed to the onClick and startClick callback functions. This allows you to associate custom data with the card (e.g., an ID, object reference, or metadata) that can be accessed when the card is clicked. The data is included in the callback payload as { event: MouseEvent, data: any }.
iconRight string undefined Optional icon identifier displayed on the right side of the card. Should match an available icon in your icon library. The icon is displayed as a button (16px size) with red color. When clicked, it triggers the startClick callback (if provided) and stops event propagation to prevent triggering the card's onClick handler. The icon's vertical position can be controlled with iconPosition.
iconPosition 'bottom' | 'center' | 'top' 'bottom' Controls the vertical alignment of the right icon and the main content area. Options:
  • 'top': Aligns content to the top
  • 'center': Centers content vertically
  • 'bottom': Aligns content to the bottom (default)
This affects the flex alignment of the content container.
className string '' Optional additional CSS classes to apply to the card container. Useful for custom styling, borders, backgrounds, or layout adjustments (e.g., 'border-l-2 border-red-500 rounded-none'). The component has default styling with padding, background, rounded corners, and flex layout.
iconPositionClassName string '' Optional additional CSS classes to apply specifically to the right icon button. Useful for custom styling the icon (e.g., color, size, positioning). Example: '!text-amber-500' to change the icon color to amber. This allows fine-grained control over the right icon's appearance without affecting other elements.
removeImageRounded boolean false When true, removes the rounded (circular) styling from the image, making it square with small rounded corners (rounded-sm). When false, the image is circular (rounded-full). This also affects the label styling: when true, the label is bold and dark; when false, it's normal weight and gray.
onClick (item: any) => void undefined Optional callback function executed when the card is clicked. The function receives an object with event (the MouseEvent) and data (the value of the data input): { event: MouseEvent, data: any }. This is triggered by clicking anywhere on the card except the right icon button (which uses startClick). Use to handle card navigation, selection, or other click actions.
startClick (item: any) => void undefined Optional callback function executed when the right icon button is clicked. The function receives an object with event (the MouseEvent) and data (the value of the data input): { event: MouseEvent, data: any }. The click event propagation is stopped to prevent triggering the card's onClick handler. Use to handle icon-specific actions (e.g., start, play, edit) that are separate from the main card click action.

Additional Notes:

  • Content Projection: The component supports content projection using ng-content. You can place any HTML content between the opening and closing tags of <ds-sheet-card>, and it will be rendered below the label in the main content area. This allows for flexible card content (e.g., descriptions, additional text, or custom elements).
  • Left Content Priority: The left side of the card displays content in the following priority order:
    1. leftContent (TemplateRef or string) - highest priority
    2. image (if provided)
    3. icon (if provided)
    Only one of these will be displayed at a time.
  • Image Styling: Images are displayed in a 40px × 40px container with object-fit: cover to maintain aspect ratio. By default, images are circular, but this can be changed with removeImageRounded. The image uses the label as its alt text for accessibility.
  • Icon Styling: Icons (both left and right) are displayed with red color by default. The left icon is in a circular container with white background and border. The right icon is a standalone button. Both use the icon library component (ds-icon).
  • Draggable Functionality: When isDraggable is set to 'true', the card:
    • Shows a drag handle icon on hover at the left side
    • Adjusts padding on hover to accommodate the drag handle
    • Uses smooth transitions for hover effects
    Note: The component only provides the visual indicator; actual drag-and-drop functionality must be implemented separately (e.g., using Angular CDK drag-drop or a custom implementation).
  • Click Handling: The component has two separate click handlers:
    • Card click (onClick): Triggered when clicking anywhere on the card except the right icon button
    • Icon click (startClick): Triggered when clicking the right icon button, with event propagation stopped
    This allows for different actions based on where the user clicks.
  • Label Styling: The label appearance depends on removeImageRounded:
    • When removeImageRounded: true: Label is bold and dark (font-bold text-neutral-900)
    • When removeImageRounded: false: Label is normal weight and gray (font-normal text-neutral-500)
  • Custom Left Content: When using leftContent with a TemplateRef, you can create complex custom layouts. For example, you can display multiple icons, custom graphics, or interactive elements. The template is rendered using ngTemplateOutlet, giving you full control over the left side content.
  • Responsive Design: The card uses flexbox for layout, making it responsive. The content area takes up the available space (w-full), and the right icon is positioned at the end. The card maintains its structure across different screen sizes.
  • Default Styling: The card has default styling with:
    • Padding: px-3.5 py-3
    • Background: bg-level-3
    • Rounded corners: rounded
    • Flex layout: flex flex-row items-center
    • Gap: gap-2 between elements
    These can be overridden or extended using the className input.
