Account Slider Card

# API Reference

Property Type Default Description
Component Inputs
account_number string - Required. The account number or identifier displayed on the card. This is the primary identifier for the account.
balance number - Required. The account balance amount displayed on the card. Should be a numeric value representing the current balance. Can be masked when isDiscrete is true.
currency string - Required. The currency code (e.g., 'USD', 'EUR', 'GBP') displayed alongside the balance. Used for formatting the balance display.
type string - Required. The account type (e.g., 'Savings', 'Checking', 'Credit'). Displayed on the card to categorize the account.
isDiscrete boolean false When true, masks or hides sensitive financial information (like the balance and account number) for privacy. Use when displaying accounts in public or shared environments.
id string undefined Optional unique identifier for the account. Used internally for tracking and can be passed to action handlers via the data property.
actionsList ICallToActionIcon[] [] Array of action icons displayed on the card (e.g., edit, delete, more options). Each action can have an icon, click handler, and optional dropdown items. See the ICallToActionIcon Interface section below for details.
minHeight number 152 Minimum height in pixels for the card. Useful for maintaining consistent card heights in slider or grid layouts. Defaults to 152px.
data any undefined Optional data object passed to action handlers and the card click event. When action icons are clicked or the card is clicked, this data is included in the event payload, allowing handlers to access the full account information.
className string '' Optional additional CSS classes to apply to the card container. Useful for custom styling or layout adjustments (e.g., setting max width: max-w-[275px]).
Component Outputs (Events)
onClick EventEmitter<any> - Emitted when the card is clicked. Provides an object with data and event properties. The data property contains the value passed to the card's data input. Use for navigation, selection, or any card-level interactions in slider or carousel contexts.

# ICallToActionIcon Interface

The ICallToActionIcon interface defines the structure for action items passed to the actionsList input property.

Property Type Default Description
icon string - Required. The icon identifier or name to display. Should match an available icon in your icon library (e.g., 'pen', 'trash', 'more-vertical').
onClick (item?: any) => void undefined Optional callback function executed when the action icon is clicked. Receives an object with event and data properties. The data property contains the value passed to the card's data input.
buttonClassName string undefined Optional CSS classes to apply to the action button. Use for custom styling of individual action icons (e.g., different colors for different actions).
dropdownItems DropdownItem[] undefined Optional array of dropdown menu items. When provided, clicking the action icon opens a dropdown menu with these items instead of directly executing the onClick handler. Useful for actions with multiple options (e.g., a "more" menu).
