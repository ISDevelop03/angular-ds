API Reference

Property Type Default Description
Component Inputs
title string - Required. The title or name of the account grid card displayed in the header. This is the primary text label for the card.
price number - Required. The price or balance amount displayed on the card. Shown with currency formatting and can be masked when isDiscrete is true.
currency string - Required. The currency code (e.g., 'USD', 'EUR', 'GBP') displayed alongside the price. Used for formatting the balance display.
dateOperation string - Required. The operation date displayed on the card. Represents the date when the transaction or operation occurred. Format as needed for display (e.g., 'YYYY-MM-DD' or formatted date string).
dateValue string - Required. The value date displayed on the card. Represents the date when the transaction value is effective or settled. Format as needed for display (e.g., 'YYYY-MM-DD' or formatted date string).
isDiscrete boolean false When true, masks or hides sensitive financial information (like the price/balance) for privacy. Use when displaying accounts in public or shared environments. The balance formatter component handles the masking logic.
className string '' Optional additional CSS classes to apply to the card container. Useful for custom styling or layout adjustments.
Content Projection
ng-content - - The component supports content projection for an expandable accordion panel. Any content placed between the opening and closing tags will be displayed in a collapsible section below the card header. The accordion can be toggled programmatically or through user interaction. Use for displaying additional account details, transactions, or any custom content.
