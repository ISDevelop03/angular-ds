Account Holding Selected Card

# API Reference

Property Type Default Description
Component Inputs
id any - Optional unique identifier for the account holding selected card. Used internally for tracking and can be passed to action handlers via the data property.
image string - Required. The image URL or path for the account holding. Displayed on the card, typically as an avatar or logo. If no image is provided, the component may display initials derived from the title.
title string - Required. The title or name of the account holding displayed on the card. This is the primary text label for the card. If no image is provided, the first two letters of the title are used to generate an avatar.
actionsList ICallToActionIcon[] [] Array of action icons displayed on the card (e.g., edit, delete, more options). Actions are displayed in a dropdown menu triggered by a "more" icon in the top-right corner. Each action can have an icon, click handler, and optional dropdown items. See the ICallToActionIcon Interface section below for details.
data any undefined Optional data object passed to action handlers and the card click event. When action icons are clicked or the card is clicked, this data is included in the event payload, allowing handlers to access the full account holding information.
className string '' Optional additional CSS classes to apply to the card container. Useful for custom styling or layout adjustments (e.g., setting max width: max-w-[270px]).
Component Outputs (Events)
onClick EventEmitter<any> - Emitted when the card is clicked. Provides an object with data and event properties. The data property contains the value passed to the card's data input. Use for navigation, opening details, or any card-level interactions. The entire card is clickable, making it suitable for selection scenarios.

# ICallToActionIcon Interface

The ICallToActionIcon interface defines the structure for action items passed to the actionsList input property.

Property Type Default Description
icon string - Required. The icon identifier or name to display. Should match an available icon in your icon library (e.g., 'pen', 'trash', 'more-vertical').
onClick (item?: any) => void undefined Optional callback function executed when the action icon is clicked. Receives an object with event and data properties. The data property contains the value passed to the card's data input.
buttonClassName string undefined Optional CSS classes to apply to the action button. Use for custom styling of individual action icons (e.g., different colors for different actions).
dropdownItems DropdownItem[] undefined Optional array of dropdown menu items. When provided, clicking the action icon opens a dropdown menu with these items instead of directly executing the onClick handler. Useful for actions with multiple options (e.g., a "more" menu).
