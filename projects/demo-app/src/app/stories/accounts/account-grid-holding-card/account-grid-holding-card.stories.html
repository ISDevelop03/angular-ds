API Reference

Property Type Default Description
Component Inputs
id any - Required. Unique identifier for the account grid holding card. Used internally for tracking and can be passed to action handlers via the data property.
title string - Required. The title or name of the account grid holding displayed on the card. This is the primary text label for the card.
accountNumber number - Required. The account number displayed on the card. Can be copied to clipboard if copyToClipboard function is provided.
currency string - Required. The currency code (e.g., 'USD', 'EUR', 'GBP') displayed alongside the balance amounts. Used for formatting the balance displays.
soldeComptable number - Required. The accounting balance (book balance) displayed on the card. Used for financial reporting and may differ from the real-time balance. Can be masked when isDiscrete is true.
soldeReel number - Required. The real-time balance displayed on the card. Represents the current available balance including pending transactions, which may differ from the accounting balance. Can be masked when isDiscrete is true.
isDiscrete boolean false When true, masks or hides sensitive financial information (like balances and account numbers) for privacy. Use when displaying accounts in public or shared environments.
actionsList ICallToActionIcon[] [] Optional array of action icons displayed on the card (e.g., edit, delete, more options). The component automatically adds an 'arrow-down' icon to toggle the accordion panel. Each action can have an icon, click handler, and optional dropdown items. See the ICallToActionIcon Interface section below for details.
editLink string undefined Optional URL or route path for editing the account. When provided, enables navigation to an edit page or form for the account.
copyToClipboard (accountNumber: number) => void undefined Optional callback function for copying the account number to the clipboard. When provided, enables a copy functionality for the account number. The function receives the account number as a parameter.
data any undefined Optional data object passed to action handlers. When action icons are clicked, this data is included in the event payload, allowing handlers to access the full account information.
className string '' Optional additional CSS classes to apply to the card container. Useful for custom styling or layout adjustments.
Content Projection
ng-content - - The component supports content projection for an expandable accordion panel. Any content placed between the opening and closing tags will be displayed in a collapsible section below the card header. Click the arrow-down icon (automatically added to actionsList) to toggle the panel visibility. Use for displaying additional account details, transactions, or any custom content.

ICallToActionIcon Interface

The ICallToActionIcon interface defines the structure for action items passed to the actionsList input property.

Property Type Default Description
icon string - Required. The icon identifier or name to display. Should match an available icon in your icon library (e.g., 'pen', 'trash', 'more-vertical').
onClick (item?: any) => void undefined Optional callback function executed when the action icon is clicked. Receives an object with event and data properties. The data property contains the value passed to the card's data input.
buttonClassName string undefined Optional CSS classes to apply to the action button. Use for custom styling of individual action icons (e.g., different colors for different actions).
dropdownItems DropdownItem[] undefined Optional array of dropdown menu items. When provided, clicking the action icon opens a dropdown menu with these items instead of directly executing the onClick handler. Useful for actions with multiple options (e.g., a "more" menu). Each dropdown item's onClick handler receives the card's data in the event payload.
