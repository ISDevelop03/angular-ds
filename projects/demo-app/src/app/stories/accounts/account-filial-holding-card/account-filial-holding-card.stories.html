API Reference

Property Type Default Description
Component Inputs
id any - Required. Unique identifier for the account filial holding card. Used internally for tracking and can be passed to action handlers via the data property.
title string - Required. The title or name of the account filial holding displayed on the card. This is the primary text label for the card.
image string - Required. The image URL or path for the account filial holding. Displayed on the card, typically as an avatar or logo. If no image is provided, the component may display initials derived from the title.
actionsList ICallToActionIcon[] [] Optional array of action icons displayed on the card (e.g., edit, delete, more options). The component automatically adds an 'arrow-down' icon to toggle the accordion panel. Each action can have an icon, click handler, and optional dropdown items. See the ICallToActionIcon Interface section below for details.
data any undefined Optional data object passed to action handlers. When action icons are clicked, this data is included in the event payload, allowing handlers to access the full account filial holding information.
className string '' Optional additional CSS classes to apply to the card container. Useful for custom styling or layout adjustments.
Content Projection
ng-content - - The component supports content projection. Any content placed between the opening and closing tags will be displayed in an expandable accordion panel. Click the arrow-down icon to toggle the panel visibility. Use for displaying additional account details, transactions, or any custom content.

ICallToActionIcon Interface

The ICallToActionIcon interface defines the structure for action items passed to the actionsList input property.

Property Type Default Description
icon string - Required. The icon identifier or name to display. Should match an available icon in your icon library (e.g., 'pen', 'trash', 'more-vertical').
onClick (item?: any) => void undefined Optional callback function executed when the action icon is clicked. Receives an object with event and data properties. The data property contains the value passed to the card's data input.
buttonClassName string undefined Optional CSS classes to apply to the action button. Use for custom styling of individual action icons (e.g., different colors for different actions).
dropdownItems DropdownItem[] undefined Optional array of dropdown menu items. When provided, clicking the action icon opens a dropdown menu with these items instead of directly executing the onClick handler. Useful for actions with multiple options (e.g., a "more" menu).
