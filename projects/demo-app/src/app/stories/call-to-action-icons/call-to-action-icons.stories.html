# API Reference

Property Type Default Description
Component Inputs
actions ICallToActionIcon[] [] Required. Array of action icon objects to display. Each action represents an icon button or dropdown. Actions are rendered horizontally with separators between them (depending on the variant). See the ICallToActionIcon Interface section below for details.
variant keyof typeof callToActionIconsTheme 'default' Visual style variant of the call-to-action icons container. Available options: 'default', 'separated', 'borderless'. Each variant has distinct wrapper, button, separator, and icon styling. The 'default' variant shows a border around all icons with vertical separators. The 'separated' variant shows each icon in its own bordered container. The 'borderless' variant shows icons without borders or separators.
className string '' Optional additional CSS classes to apply to the call-to-action icons wrapper. Useful for custom styling or layout adjustments (e.g., w-fit for width control).

# ICallToActionIcon Interface

The ICallToActionIcon interface defines the structure for action items passed to the actions input property.

Property Type Default Description
icon string - Required. Icon identifier displayed in the action button. Should match an available icon in your icon library (e.g., 'edit', 'delete', 'more', 'download'). The icon size is determined by the variant theme (default: 16px).
onClick (item?: any) => void undefined Optional callback function executed when the action button is clicked. Receives the click event as a parameter. Only applies when dropdownItems is not provided (simple button action, not a dropdown). Use to handle click events and perform actions.
buttonClassName string undefined Optional CSS classes to apply to the individual action button. Use for custom styling of specific action buttons. Only applies when dropdownItems is not provided (simple button, not a dropdown).
dropdownItems DropdownItem[] undefined Optional array of dropdown items. When provided, the action renders as a dropdown menu instead of a simple button. The icon becomes a trigger for the dropdown. When not provided, the action renders as a clickable button. Mutually exclusive with onClick (use dropdown item handlers instead).
