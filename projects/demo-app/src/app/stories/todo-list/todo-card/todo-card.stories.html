Todo Card

TodoCard

# API Reference

Property Type Default Description
Component Inputs
title string '' Required. The title text displayed at the top of the todo card. Shown in bold text with a font size of 0.875rem (14px) and line height of 16px. This is the main heading for the todo item (e.g., "Paiement de facture", "Virement de masse").
description string '' Optional description text displayed below the title. Shown in smaller text (0.75rem / 12px) with gray color. This provides additional context or details about the todo item. The description is only rendered if provided (conditional rendering with *ngIf).
date string '' Required. The due date for the todo item. Can be provided as a date string (e.g., "2025-02-26") or any format recognized by Angular's date pipe. The date is formatted and displayed as "Date d'échéance : dd.MM.yyyy" (e.g., "Date d'échéance : 26.02.2025") with a clock icon. The date is only rendered if provided (conditional rendering with *ngIf).
badgeText string '' Required. The text content displayed in the badge. This is the status label for the todo item (e.g., "En attente de paiement", "En attente de signature"). The badge is displayed with pill styling (rounded-full) and small size. The badge is only rendered if badgeText is provided (conditional rendering with *ngIf).
badgeVariant string '' Required. Visual style variant for the badge. Available options: 'default', 'amber', 'blue', 'green', 'violet', 'red', 'orange'. Each variant has a distinct background and text color combination. Common usage: 'red' for urgent/important items, 'amber' for pending items, 'green' for completed items, etc.
icon string '' Optional icon identifier displayed on the left side of the card. Should match an available icon in your icon library (e.g., 'message', 'flag', 'file', 'book'). The icon is displayed in a circular container (36px × 36px) with a border, background, and red color. The icon size is 18px. The icon is only rendered if provided (conditional rendering with *ngIf).
isRead boolean false When true, indicates that the todo item has been read. Read items have different styling:
  • Left border: border-l border-red-500 (red left border)
  • Background: bg-card-level-6 (darker background)
When false (unread), the card has:
  • Bottom border: border-b border-stroke-1
  • Background: bg-level-4 (lighter background)
Use to visually distinguish read vs unread todo items.
className string '' Optional additional CSS classes to apply to the card container. Useful for custom styling, spacing, or layout adjustments. The component has default styling with flex layout, padding, gap spacing, and conditional borders/backgrounds based on isRead.
