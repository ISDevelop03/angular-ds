Todo Card

TodoCard

# API Reference

Property Type Default Description
Component Inputs
title string '' Required. The title text displayed at the top of the todo card. Shown in bold text with a font size of 0.875rem (14px) and line height of 16px. This is the main heading for the todo item (e.g., "Paiement de facture", "Virement de masse").
description string '' Optional description text displayed below the title. Shown in smaller text (0.75rem / 12px) with gray color. This provides additional context or details about the todo item. The description is only rendered if provided (conditional rendering with *ngIf).
date string '' Required. The due date for the todo item. Can be provided as a date string (e.g., "2025-02-26") or any format recognized by Angular's date pipe. The date is formatted and displayed as "Date d'échéance : dd.MM.yyyy" (e.g., "Date d'échéance : 26.02.2025") with a clock icon. The date is only rendered if provided (conditional rendering with *ngIf).
badgeText string '' Required. The text content displayed in the badge. This is the status label for the todo item (e.g., "En attente de paiement", "En attente de signature"). The badge is displayed with pill styling (rounded-full) and small size. The badge is only rendered if badgeText is provided (conditional rendering with *ngIf).
badgeVariant string '' Required. Visual style variant for the badge. Available options: 'default', 'amber', 'blue', 'green', 'violet', 'red', 'orange'. Each variant has a distinct background and text color combination. Common usage: 'red' for urgent/important items, 'amber' for pending items, 'green' for completed items, etc.
icon string '' Optional icon identifier displayed on the left side of the card. Should match an available icon in your icon library (e.g., 'message', 'flag', 'file', 'book'). The icon is displayed in a circular container (36px × 36px) with a border, background, and red color. The icon size is 18px. The icon is only rendered if provided (conditional rendering with *ngIf).
isRead boolean false When true, indicates that the todo item has been read. Read items have different styling:
  • Left border: border-l border-red-500 (red left border)
  • Background: bg-card-level-6 (darker background)
When false (unread), the card has:
  • Bottom border: border-b border-stroke-1
  • Background: bg-level-4 (lighter background)
Use to visually distinguish read vs unread todo items.
className string '' Optional additional CSS classes to apply to the card container. Useful for custom styling, spacing, or layout adjustments. The component has default styling with flex layout, padding, gap spacing, and conditional borders/backgrounds based on isRead.

Additional Notes:

  • Card Layout: The todo card uses a horizontal flex layout with:
    • Left side: Optional icon in a circular container
    • Right side: Content area with title, description, badge, and date
    The card has padding (p-2) and gap spacing (gap-2) between elements.
  • Read/Unread States: The component provides visual distinction between read and unread items:
    • Unread (isRead: false): Bottom border, lighter background, appears more prominent
    • Read (isRead: true): Red left border, darker background, appears less prominent
    This helps users quickly identify which items need attention.
  • Badge Integration: The component uses the ds-badge component internally to display the status badge. The badge is configured with:
    • pill="true" for rounded-full styling
    • size="sm" for small size
    • variant from the badgeVariant input
  • Date Formatting: The date is automatically formatted using Angular's date pipe with the format "dd.MM.yyyy" (e.g., "26.02.2025"). The date is displayed with a clock icon and the label "Date d'échéance :" (Due date:). The date pipe uses the French locale format.
  • Conditional Rendering: Several elements are conditionally rendered:
    • Icon: Only shown if icon is provided
    • Description: Only shown if description is provided
    • Badge: Only shown if badgeText is provided
    • Date: Only shown if date is provided
    This allows for flexible card layouts based on available data.
  • Icon Styling: When an icon is provided, it's displayed in a circular container with:
    • Size: 36px × 36px (size-[36px])
    • Border: border border-stroke-1
    • Background: bg-card-level-2
    • Icon color: Red (text-red-500)
    • Icon size: 18px
  • Typography: The card uses specific font sizes and weights:
    • Title: 0.875rem (14px) font size, 16px line height, bold weight
    • Description: 0.75rem (12px) font size, 16px line height, normal weight, gray color
    • Date: 0.75rem (12px) font size, 16px line height, normal weight, gray color
  • Spacing: The card uses consistent spacing:
    • Card padding: p-2 (8px)
    • Gap between icon and content: gap-2 (8px)
    • Gap between content elements: gap-3 (12px)
  • Use Cases: This component is designed for displaying todo items, tasks, or notifications in a list format, such as:
    • Task lists
    • Notification cards
    • Action items
    • Pending approvals
    • Reminders or alerts
  • Badge Variants: The badge variant determines the visual appearance of the status badge. Common usage patterns:
    • 'red': Urgent items, errors, or critical status
    • 'amber': Pending items, warnings, or in-progress status
    • 'green': Completed items or success status
    • 'blue': Informational items or neutral status
    • 'violet': Special or custom status
    • 'orange': High priority or attention-required status
  • Responsive Design: The card uses flexbox for layout, making it responsive. The content area takes up the available space, and the icon (if provided) is positioned on the left. The card maintains its structure across different screen sizes.
