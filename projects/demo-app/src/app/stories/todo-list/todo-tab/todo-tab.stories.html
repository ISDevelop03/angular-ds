TodoTab

# API Reference

Property Type Default Description
Component Inputs
title string '' Required. The title text displayed below the count and icon. Shown in small text (0.75rem / 12px) with medium font weight. This is the label for the todo tab (e.g., "Mes opérations en attente", "Tâches administratives").
count number 0 The number displayed prominently at the top of the tab. Shown in large text (2rem / 32px) with bold font weight and gray color. This typically represents the number of items or tasks in the category (e.g., 6, 4). The count is always displayed, even if 0.
icon string '' Optional icon identifier displayed next to the count. Should match an available icon in your icon library (e.g., 'flag', 'message', 'file'). The icon is displayed in red color (text-red-500) with a size of 20px. The icon is only rendered if provided (conditional rendering with *ngIf).
isOpen boolean false When true, indicates that the tab is in an open/active state. Open tabs have:
  • Red bottom border: border-red-500
  • Arrow icon rotated 180 degrees and colored red
When false (closed), the tab has:
  • Transparent bottom border: border-transparent
  • Arrow icon in default position and color
Use to control the visual state of the tab and indicate which section is currently active or expanded.
onClick () => void undefined Optional callback function executed when the tab is clicked. The function receives no parameters. Use to handle tab clicks, toggle the isOpen state, or perform other actions when the user interacts with the tab. The callback is only executed if provided (conditional execution with onClick && onClick()).
className string '' Optional additional CSS classes to apply to the tab container. Useful for custom styling, spacing, or layout adjustments. The component has default styling with flex layout, padding, gap spacing, and conditional borders based on isOpen.

Additional Notes:

  • Tab Layout: The todo tab uses a vertical flex layout with:
    • Top section: Count number and optional icon displayed in a horizontal flex container
    • Bottom section: Title text and arrow icon displayed in a horizontal flex container with space-between alignment
    The tab has padding (py-3) and gap spacing (gap-2) between sections.
  • Open/Closed States: The component provides visual distinction between open and closed states:
    • Open (isOpen: true): Red bottom border, arrow rotated 180 degrees and colored red, appears active
    • Closed (isOpen: false): Transparent bottom border, arrow in default position, appears inactive
    The transitions are animated with transition-all duration-500 ease-in-out for smooth state changes.
  • Count Display: The count is displayed prominently at the top:
    • Font size: 2rem (32px)
    • Line height: 32px
    • Font weight: Bold
    • Color: Gray (text-neutral-600)
    The count is always displayed, even if 0, allowing users to see when there are no items in a category.
  • Icon Display: When an icon is provided, it's displayed next to the count:
    • Icon size: 20px
    • Icon color: Red (text-red-500)
    • Gap between count and icon: gap-2 (8px)
    The icon helps visually identify the category or type of items in the tab.
  • Arrow Icon: The component always displays an "arrow-down" icon at the bottom right:
    • When closed: Arrow points down in default color
    • When open: Arrow is rotated 180 degrees (points up) and colored red
    The rotation and color change are animated with transitions, providing visual feedback about the tab's state.
  • Click Handling: The tab is clickable and has:
    • tabindex="0" for keyboard accessibility
    • role="button" for screen readers
    • Click handler that executes onClick if provided
    The click handler is conditionally executed, so the tab can be used with or without an onClick callback.
  • Typography: The tab uses specific font sizes and weights:
    • Count: 2rem (32px) font size, 32px line height, bold weight, gray color
    • Title: 0.75rem (12px) font size, 16px line height, medium weight, neutral-900 color
  • Spacing: The tab uses consistent spacing:
    • Vertical padding: py-3 (12px top and bottom)
    • Gap between sections: gap-2 (8px)
    • Gap between count and icon: gap-2 (8px)
    • Gap between title and arrow: gap-2 (8px)
  • Border Styling: The tab has a bottom border that changes based on state:
    • Open: border-b border-red-500 (red bottom border, 1px)
    • Closed: border-b border-transparent (transparent bottom border, maintains layout)
    The border transition is animated for smooth state changes.
  • Use Cases: This component is designed for displaying todo categories or tabs in a todo list interface, such as:
    • Todo list categories (e.g., "Pending", "In Progress", "Completed")
    • Task groups with item counts
    • Filterable sections in a todo application
    • Expandable/collapsible todo categories
  • State Management: The component doesn't manage its own state internally. You need to:
    • Control the isOpen state externally
    • Provide an onClick callback to toggle the state
    • Update the isOpen value when the tab is clicked
    This allows for flexible state management and integration with parent components or state management libraries.
  • Accessibility: The component includes accessibility features:
    • tabindex="0" for keyboard navigation
    • role="button" for screen readers
    • Click handler for mouse and keyboard interactions
  • Responsive Design: The tab uses flexbox for layout, making it responsive. The content adapts to the available width, and the title and arrow maintain their alignment across different screen sizes.
  • Multiple Tabs: When using multiple tabs side by side, you typically:
    • Place them in a flex container with horizontal layout
    • Add separators between tabs (e.g., vertical dividers)
    • Manage the isOpen state for each tab independently
    • Ensure only one tab is open at a time (if using accordion-style behavior)
