# API Reference

Property Type Default Description
Component Inputs
showOptions boolean true When true, displays debit/credit radio options in the dropdown menu. When false, only the price input field is shown. This also affects the output format of the onSelect event.
className string '' Optional additional CSS classes to apply to the root container element. Useful for custom styling or layout adjustments.
value number | null null The current price value to display (controlled/bound value). When set, the trigger shows the formatted price and the input is pre-filled. Use null to show the placeholder and allow the user to enter a new value. Updates to value are reflected in the displayed price and placeholder.
Component Outputs (Events)
onSelect EventEmitter<{ type: string, price: number } | number> - Emitted when the user clicks the "Valider" button. When showOptions is true, emits an object with { type: 'debit' | 'credit', price: number }. When showOptions is false, emits only the price as a number.
Behavior & Features
Price Formatting - - The input automatically formats numbers with space as thousands separator and comma as decimal separator (e.g., 1 234,56). Maximum 2 decimal places allowed.
Dropdown Animation - - The dropdown menu uses Angular animations with scale and opacity transitions (120ms ease-out on enter, 100ms ease-in on leave).
Auto-positioning - - The dropdown automatically adjusts its position to stay within the viewport boundaries. It uses Angular CDK Portal to render at document body level, preventing overflow issues.
Outside Click - - The dropdown closes automatically when clicking outside the trigger button or menu area.
Clear Selection - - After a price is selected, a close icon appears in the trigger button allowing users to clear the current selection and reset to the default placeholder.
