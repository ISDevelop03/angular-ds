# API Reference

Property Type Default Description
Component Inputs
checked boolean false The checked/on state of the toggle switch. When true, the toggle is in the enabled/on position (green background, switch moved to the right). When false, the toggle is in the disabled/off position (gray background, switch moved to the left). The component automatically syncs its internal enabled state when this value changes via ngOnChanges.
disabled boolean false When true, disables the toggle switch, preventing user interaction. Disabled toggles have:
  • Reduced opacity: opacity-50
  • Not-allowed cursor: cursor-not-allowed
  • Click handler is ignored (no state change occurs)
Use for toggles that should be temporarily unavailable or require certain conditions to be met.
variant keyof typeof ToggleTheme 'default' Theme variant for styling the toggle switch. Currently only 'default' is available. The variant determines the visual styling, colors, and animations of the toggle switch. Future variants may be added for different color schemes or styles.
className string '' Optional additional CSS classes to apply to the toggle switch container. Useful for custom styling, spacing, or layout adjustments. The component has default styling with relative positioning, flex layout, rounded-full shape, and specific dimensions (36px width, 20px height).
Component Outputs (Events)
onChange EventEmitter<boolean> - Emitted when the toggle state changes. The event is emitted in two scenarios:
  • User interaction: When the user clicks the toggle (if not disabled), the new state (true or false) is emitted.
  • Programmatic change: When the checked input changes via ngOnChanges, the new state is emitted.
The payload is a boolean value representing the new checked state (true for enabled/on, false for disabled/off). Use to handle toggle state changes, update parent component state, or perform actions based on the toggle state.

Additional Notes:

  • Toggle Layout: The toggle switch uses a horizontal layout with:
    • Container: 36px width, 20px height, rounded-full shape
    • Switch circle: 17px Ã— 17px, rounded-full, moves horizontally based on state
    The switch circle translates horizontally to indicate the on/off state.
  • Enabled/Disabled States: The toggle has distinct visual states:
    • Enabled/On (checked: true): Green background (bg-success), switch circle moved to the right (translate-x-[17px])
    • Disabled/Off (checked: false): Gray background (bg-neutral-200), switch circle moved to the left (translate-x-[2px])
    The transitions are animated with transition-all duration-300 ease-in for smooth state changes.
  • State Management: The component maintains an internal enabled state that:
    • Is initialized from the checked input
    • Updates when checked changes (via ngOnChanges)
    • Toggles when the user clicks the switch (if not disabled)
    • Triggers the onChange event when it changes
    This allows for both controlled (via checked input) and uncontrolled (via user clicks) usage patterns.
  • Click Handling: The toggle is clickable and:
    • Has a pointer cursor (cursor-pointer) when enabled
    • Executes toggleChecked() when clicked
    • Only toggles if not disabled
    • Emits the onChange event with the new state
  • Animations: The toggle includes smooth animations:
    • Container: transition-all duration-300 ease-in for background color changes
    • Switch circle: transition-all duration-150 ease-in for position changes
    The animations provide visual feedback and make state changes feel smooth and responsive.
  • RTL Support: The toggle includes RTL (right-to-left) support:
    • When enabled: translate-x-[17px] rtl:-translate-x-[17px]
    • When disabled: translate-x-[2px] rtl:-translate-x-[2px]
    This ensures the toggle works correctly in both LTR and RTL layouts.
  • Styling: The toggle uses Tailwind CSS classes for styling:
    • Container: Relative positioning, inline-flex, rounded-full, specific dimensions
    • Switch circle: Inline-block, rounded-full, white background (bg-level-4), transform for positioning
    • Colors: Green for enabled, gray for disabled
  • Use Cases: This component is designed for binary on/off or enabled/disabled controls, such as:
    • Feature toggles (enable/disable features)
    • Settings switches (notifications, preferences)
    • Form controls (agree to terms, opt-in/opt-out)
    • State toggles (active/inactive, visible/hidden)
  • Two-Way Binding: You can use two-way data binding with [(checked)] to automatically sync the toggle state with a parent component property. This is useful for form controls or when you want to keep the toggle state in sync with external state.
  • Programmatic Control: The toggle can be controlled programmatically by:
    • Setting the checked input to true or false
    • The component automatically updates its internal state and emits the onChange event
    This allows for external control of the toggle state, such as resetting to a default value or syncing with server state.
  • Accessibility: The toggle is accessible and can be enhanced with:
    • Keyboard navigation (click/tap to toggle)
    • ARIA attributes (can be added via className or wrapper elements)
    • Screen reader support (via proper labeling)
    Consider adding aria-label or aria-labelledby for better accessibility.
  • Integration with Forms: The toggle can be integrated with Angular forms by:
    • Using ngModel for template-driven forms
    • Using FormControl for reactive forms
    • Binding to form control values and handling validation
    The onChange event can be used to update form control values.
