# API Reference

Property Type Default Description
Component Inputs
checked boolean false The checked/on state of the toggle switch. When true, the toggle is in the enabled/on position (green background, switch moved to the right). When false, the toggle is in the disabled/off position (gray background, switch moved to the left). The component automatically syncs its internal enabled state when this value changes via ngOnChanges.
disabled boolean false When true, disables the toggle switch, preventing user interaction. Disabled toggles have:
  • Reduced opacity: opacity-50
  • Not-allowed cursor: cursor-not-allowed
  • Click handler is ignored (no state change occurs)
Use for toggles that should be temporarily unavailable or require certain conditions to be met.
variant keyof typeof ToggleTheme 'default' Theme variant for styling the toggle switch. Currently only 'default' is available. The variant determines the visual styling, colors, and animations of the toggle switch. Future variants may be added for different color schemes or styles.
className string '' Optional additional CSS classes to apply to the toggle switch container. Useful for custom styling, spacing, or layout adjustments. The component has default styling with relative positioning, flex layout, rounded-full shape, and specific dimensions (36px width, 20px height).
Component Outputs (Events)
onChange EventEmitter<boolean> - Emitted when the toggle state changes. The event is emitted in two scenarios:
  • User interaction: When the user clicks the toggle (if not disabled), the new state (true or false) is emitted.
  • Programmatic change: When the checked input changes via ngOnChanges, the new state is emitted.
The payload is a boolean value representing the new checked state (true for enabled/on, false for disabled/off). Use to handle toggle state changes, update parent component state, or perform actions based on the toggle state.
