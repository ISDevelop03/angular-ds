# API Reference

Property Type Default Description
Component Inputs
totalItems number 0 Total number of items to paginate. Together with itemsPerPage it drives pageCount = max(1, ceil(totalItems / itemsPerPage)). When pageCount is 1, the component does not render (single-page and empty lists are hidden).
itemsPerPage number perPageOptions[0] Number of items per page. Defaults to perPageOptions[0] in the component constructor. Bind with [itemsPerPage] from the parent and keep it in sync when the user picks a new size by handling (perPageChange) (there is no itemsPerPageChange output for banana-in-the-box syntax). Changing the size via the per-page control emits perPageChange and then attempts to select page 1; if the active page was already 1, pageChange is not emitted again.
currentPage number 1 Active page index (1-based). User actions update the internal page and emit pageChange with the new value. For fully controlled usage, keep the parent model in sync with that event.
variant string 'default' Theme key for layout and classes (theme[variant] in the design-system theme map). The built-in theme currently defines 'default' only; other keys must be added to the theme object to be valid at runtime.
minimized boolean false When true, displays a minimized version of the pagination component with fewer visible elements. Use for compact layouts or when space is limited.
showPerPage boolean false When true, shows a fixed "Par page:" label and a row of buttons for each value in perPageOptions. The active size is emphasized in the template. Choosing a value updates itemsPerPage, emits perPageChange, and selects page 1 (emitting pageChange only when the previous page was not already 1). The perPageLabel input is not wired in the default template.
perPageOptions number[] [25, 50, 100] Available page sizes. Shown as buttons when showPerPage is true. The first entry seeds itemsPerPage when the component is created.
display 'horizontal' | 'vertical' 'horizontal' Layout direction for the wrapper: horizontal row vs vertical stack (extra Tailwind classes are applied when vertical).
firstItem number 1 First value in the compact info segment beside the controls (parent should set it, e.g. (currentPage - 1) * itemsPerPage + 1, if you want it to match the current slice). It is not derived automatically from currentPage inside the component.
boundaryStartCount number 2 Number of page buttons to always show at the start (beginning) of the pagination. These buttons are always visible regardless of the current page. Used to provide quick access to the first pages. The actual count is limited to the total number of pages available.
boundaryEndCount number 2 Number of page buttons to always show at the end of the pagination. These buttons are always visible regardless of the current page. Used to provide quick access to the last pages. The actual count is limited to the total number of pages available.
windowSize number 1 Neighbor radius around currentPage for the middle window (clamped so it does not overlap the fixed start/end blocks). Gaps between merged segments are rendered as an ellipsis character (…).
maxPageButtons number 5 Declared for API compatibility; the visible page list is built from boundaryStartCount, boundaryEndCount, and windowSize only, so this input does not change behavior in the current implementation.
prevLabel string 'Précédent' Label text for the "Previous" button. Displayed next to the previous arrow icon. Use to customize the button text for localization or branding.
nextLabel string 'Suivant' Label text for the "Next" button. Displayed next to the next arrow icon. Use to customize the button text for localization or branding.
perPageLabel string 'Par page' Intended label for the per-page area; the stock template uses a hard-coded "Par page:" prefix instead of binding this property.
className string '' Optional additional CSS classes to apply to the pagination component wrapper. Useful for custom styling, layout adjustments, or responsive sizing. The component has default styling with flex layout, gap, and rounded corners.
Component Outputs (Events)
pageChange EventEmitter<number> - Emitted when the user navigates to a different page. Provides the new page number (1-based). Triggered when:
  • A page number button is clicked
  • The "Previous" button is clicked (decrements page)
  • The "Next" button is clicked (increments page)
Use to update your data source, fetch new data, or sync the page state with your application.
perPageChange EventEmitter<number> - Emitted when the user picks a new page size from the per-page buttons. Payload is the new itemsPerPage. The component then selects page 1, emitting pageChange only if the page actually changes. Use it to refetch data and update bound itemsPerPage in the parent.
