# API Reference

Property Type Default Description
Component Inputs
totalItems number 0 Required. Total number of items to paginate. Used to calculate the total number of pages based on itemsPerPage. The component calculates pageCount = Math.ceil(totalItems / itemsPerPage).
itemsPerPage number perPageOptions[0] Number of items displayed per page. Supports two-way binding with [(itemsPerPage)]. Defaults to the first value in perPageOptions array. When changed, automatically resets to page 1 and triggers perPageChange event.
currentPage number 1 The currently active page number (1-based). Use to control which page is displayed. When a page is selected, the component emits pageChange with the new page number.
variant string 'compact' Visual style variant for the pagination component. Determines the theme configuration used for styling. Available variants include 'default' and 'compact'. Each variant has its own theme configuration for wrapper, container, buttons, and other elements.
minimized boolean false When true, displays a minimized version of the pagination component with fewer visible elements. Use for compact layouts or when space is limited.
showPerPage boolean false When true, displays the "items per page" selector dropdown. Allows users to change how many items are displayed per page. The selector shows options from perPageOptions with the label from perPageLabel.
perPageOptions number[] [25, 50, 100] Array of numbers representing available items-per-page options. Displayed in the per-page selector dropdown when showPerPage is true. The first value is used as the default itemsPerPage if not explicitly set.
firstItem number 1 The index of the first item displayed on the current page (1-based). Used for displaying item range information (e.g., "1-25 of 100"). Calculated as (currentPage - 1) * itemsPerPage + 1.
boundaryStartCount number 2 Number of page buttons to always show at the start (beginning) of the pagination. These buttons are always visible regardless of the current page. Used to provide quick access to the first pages. The actual count is limited to the total number of pages available.
boundaryEndCount number 2 Number of page buttons to always show at the end of the pagination. These buttons are always visible regardless of the current page. Used to provide quick access to the last pages. The actual count is limited to the total number of pages available.
windowSize number 1 Number of page buttons to show on each side of the current page (neighbors). Controls the size of the sliding window around the current page. For example, with windowSize = 1 and currentPage = 5, pages 4 and 6 are shown. Ellipses (...) are inserted when there are gaps between page ranges.
maxPageButtons number 5 Maximum number of page number buttons to display (excluding prev/next buttons and ellipses). Used to limit the total number of visible page buttons in the pagination component.
prevLabel string 'Précédent' Label text for the "Previous" button. Displayed next to the previous arrow icon. Use to customize the button text for localization or branding.
nextLabel string 'Suivant' Label text for the "Next" button. Displayed next to the next arrow icon. Use to customize the button text for localization or branding.
perPageLabel string 'Par page' Label text for the "items per page" selector. Displayed before the dropdown when showPerPage is true. Use to customize the label for localization or branding.
className string '' Optional additional CSS classes to apply to the pagination component wrapper. Useful for custom styling, layout adjustments, or responsive sizing. The component has default styling with flex layout, gap, and rounded corners.
Component Outputs (Events)
pageChange EventEmitter<number> - Emitted when the user navigates to a different page. Provides the new page number (1-based). Triggered when:
  • A page number button is clicked
  • The "Previous" button is clicked (decrements page)
  • The "Next" button is clicked (increments page)
Use to update your data source, fetch new data, or sync the page state with your application.
perPageChange EventEmitter<number> - Emitted when the user changes the number of items per page via the selector dropdown. Provides the new items-per-page value. When this event is triggered, the component automatically resets to page 1 and recalculates the total number of pages. Use to update your data source, fetch new data with the new page size, or sync the items-per-page state with your application.
