# API Reference

Property Type Default Description
ds-steps Component Inputs
currentStepIndex number 0 The index of the currently visible step. This is a zero-based index (0 for the first step, 1 for the second, etc.). The component automatically updates this value when navigating between steps using the goNext and goPrev methods. You can also control it externally to programmatically navigate to a specific step.
variant string 'default' Theme variant key for styling the steps component. Common values include 'default', 'primary', 'success', 'vertical', etc. The variant determines the visual appearance and layout of the steps indicator. For example, 'vertical' displays steps in a vertical layout.
title string '' Optional title text displayed at the top of the steps component. This is typically used to describe the overall process or workflow (e.g., "Paiement de factures", "Account Setup", "Order Process").
showNavigation boolean true When true, displays the step navigation indicator (the visual representation of steps with numbers/checkmarks). When false, hides the navigation indicator while still allowing step content to be displayed. Useful for custom navigation implementations or minimal UI designs.
showHeader boolean true When true, displays the header section of the steps component (typically containing the title). When false, hides the header section. Useful for layouts where the title is displayed elsewhere or when a minimal header is desired.
ds-step Component Inputs
title string '' The title text displayed for this step in the steps indicator. This is shown in the step header/navigation area to identify the step (e.g., "Step 1", "Step 2", "Finish").
shouldGoNext boolean true When true, allows navigation to the next step when the "Next" button is clicked. When false, prevents advancing to the next step, even if the button is clicked. This is useful for validation - set to false until the step's requirements are met (e.g., form validation, data completion). The button's onClick handler will still be called, but the step won't advance.
shouldGoPrev boolean true When true, allows navigation to the previous step when the "Prev" button is clicked. When false, prevents going back to the previous step. This is useful for steps that should not allow backward navigation (e.g., final confirmation steps). The button's onClick handler will still be called, but the step won't go back.
data any undefined Optional data object associated with this step. This data is included in the nextStep and prevStep event payloads, allowing you to pass step-specific information when navigating. Useful for storing form data, step state, or other contextual information.
leftButtons StepButton[] [{ variant: 'gray', content: 'Prev', onClick: ... }] Array of buttons displayed on the left side of the step footer. Typically used for "Previous" or "Cancel" actions. Each button is a StepButton object. The default includes a "Prev" button. See the StepButton Interface section below for details.
rightButtons StepButton[] [{ variant: 'primary', content: 'Next', pill: true, onClick: ... }] Array of buttons displayed on the right side of the step footer. Typically used for "Next", "Continue", or "Submit" actions. Each button is a StepButton object. The default includes a "Next" button with primary variant and pill styling. See the StepButton Interface section below for details.
ds-step Component Outputs (Events)
nextStep EventEmitter<{ button: StepButton; event: any; data: any; currentIndex: number }> - Emitted when advancing to the next step. The event is only emitted if shouldGoNext is true and there is a next step available. The payload contains:
  • button: The StepButton that triggered the navigation
  • event: The original click event
  • data: The step's data object
  • currentIndex: The new step index after navigation
Use to handle step completion, save data, or perform actions when moving forward.
prevStep EventEmitter<{ button: StepButton; data: any; event: any; currentIndex: number }> - Emitted when going back to the previous step. The event is only emitted if shouldGoPrev is true and there is a previous step available. The payload contains:
  • button: The StepButton that triggered the navigation
  • data: The step's data object
  • event: The original click event
  • currentIndex: The new step index after navigation
Use to handle step cancellation, restore previous state, or perform actions when moving backward.

# StepButton Interface

The StepButton interface defines the structure for buttons passed to the leftButtons and rightButtons input properties of ds-step.

Property Type Default Description
variant string - Required. Button variant for styling (e.g., 'primary', 'secondary', 'gray', 'outline'). Determines the button's visual appearance, colors, and styling. Common variants match the button component's theme variants.
content string - Required. The text content displayed on the button (e.g., "Next", "Previous", "Submit", "Cancel").
disableHandleStep boolean undefined When true, disables the automatic step navigation handling. The button's onClick handler will be called, but the component won't automatically advance or go back to steps. Use this for custom buttons that need to perform actions without step navigation (e.g., "Save Draft", "Cancel", custom workflows).
onClick (data: any) => void - Required. Callback function executed when the button is clicked. The function receives an object with: { button: StepButton, event: Event, data: any, currentIndex: number }. This handler is always called, regardless of step navigation. Use to handle button-specific actions, validation, or custom logic.
icon string undefined Optional icon identifier displayed in the button. Should match an available icon in your icon library (e.g., 'arrow-right', 'checkmark', 'close').
iconRight string undefined Optional icon identifier displayed on the right side of the button text. Useful for directional indicators (e.g., 'arrow-right' for "Next" buttons).
iconLeft string undefined Optional icon identifier displayed on the left side of the button text. Useful for directional indicators (e.g., 'arrow-left' for "Previous" buttons).
type 'button' | 'submit' | 'reset' undefined Optional HTML button type. Use 'submit' for form submission buttons, 'reset' for reset buttons, or 'button' (default) for regular action buttons.
className string undefined Optional additional CSS classes to apply to the button. Useful for custom styling, spacing, or layout adjustments.
size string undefined Optional button size (e.g., 'small', 'medium', 'large'). Determines the button's padding, font size, and overall dimensions.
disabled boolean undefined When true, disables the button, preventing clicks and showing a disabled visual state. Useful for conditional button states based on form validation or step completion.
outline boolean undefined When true, applies outline styling to the button (transparent background with border). Useful for secondary actions or less prominent buttons.
pill boolean undefined When true, applies pill/rounded styling to the button (fully rounded corners). The default "Next" button uses this styling for a more modern appearance.
