# API Reference

Property Type Default Description
Component Inputs
label string '' The label text displayed on the left side of the total list item. Shown in small text (0.875rem / 14px) with bold font weight and neutral-900 color. This is typically a descriptive label for the total (e.g., "Total", "Subtotal", "Tax", "Grand Total").
value string '' The value text displayed on the right side of the total list item, next to the icon. Shown in larger text (1.25rem / 20px) with bold font weight and neutral-900 color. This is typically a numeric value, amount, or formatted text representing the total (e.g., "100", "â‚¬1,234.56", "50%").
icon string '' The icon identifier displayed on the right side of the total list item, next to the value. Should match an available icon in your icon library (e.g., 'arrange-square', 'check', 'info'). The icon is displayed with a size of 24px and crame-500 color (text-crame-500). The icon is always displayed if provided.
className string '' Optional additional CSS classes to apply to the total list item container. Useful for custom styling, spacing, or layout adjustments. The component has default styling with padding, crame background and border, rounded corners, and flex layout with space-between alignment.

Additional Notes:

  • Card Layout: The total list item uses a horizontal flex layout with:
    • Left side: Label text displayed in a span
    • Right side: Value text and icon displayed in a paragraph with flex layout
    The layout uses justify-between to space the label and value/icon apart, with a gap of 5 units (gap-5) between them.
  • Styling: The component has default styling:
    • Background: Crame-100 (bg-crame-100)
    • Border: Crame-300, 1px (border border-crame-300)
    • Border radius: 4px (rounded-[4px])
    • Padding: py-3 px-4 (12px vertical, 16px horizontal)
    • Flex layout: flex items-center justify-between
  • Typography: The component uses specific font sizes and weights:
    • Label: 0.875rem (14px) font size, bold weight, neutral-900 color
    • Value: 1.25rem (20px) font size, bold weight, neutral-900 color
    The larger value text makes it stand out and easy to read.
  • Icon Display: When an icon is provided:
    • Icon size: 24px
    • Icon color: Crame-500 (text-crame-500)
    • Gap between value and icon: gap-1.5 (6px)
    The icon is displayed inline with the value text, providing visual context or decoration.
  • Spacing: The component uses consistent spacing:
    • Container padding: py-3 px-4 (12px vertical, 16px horizontal)
    • Gap between label and value: gap-5 (20px)
    • Gap between value and icon: gap-1.5 (6px)
  • Use Cases: This component is designed for displaying total or summary information in lists, such as:
    • Shopping cart totals (subtotal, tax, shipping, grand total)
    • Invoice summaries
    • Order summaries
    • Financial summaries
    • Statistics or metrics with labels and values
  • Value Formatting: The value is displayed as a string, so you can format it before passing it to the component:
    • Use Angular pipes for number formatting (e.g., {{ amount | number : '1.2-2' }})
    • Use currency pipes for currency formatting (e.g., {{ amount | currency : 'EUR' }})
    • Use percent pipes for percentage formatting (e.g., {{ ratio | percent }})
    • Format dates using date pipes
    The component doesn't perform any formatting itself, giving you full control over the display format.
  • Icon Usage: The icon is optional but recommended for visual enhancement:
    • Use icons that represent the type of total (e.g., currency icon for amounts, check icon for completed items)
    • Icons provide visual context and make the total list more recognizable
    • The icon color (crame-500) provides a subtle accent without overwhelming the text
  • Responsive Design: The component uses flexbox for layout, making it responsive:
    • The label and value/icon maintain their alignment across different screen sizes
    • The justify-between ensures proper spacing regardless of content length
    • Text wrapping is handled naturally by the browser
  • Multiple Items: When using multiple total list items together:
    • Place them in a flex column container with gap spacing
    • Each item maintains its own styling and spacing
    • Consider using different icons for different types of totals
    • The consistent styling creates a cohesive list appearance
  • Color Scheme: The component uses a crame color scheme:
    • Background: Crame-100 (light crame)
    • Border: Crame-300 (medium crame)
    • Icon: Crame-500 (darker crame accent)
    • Text: Neutral-900 (dark text for contrast)
    This creates a subtle, cohesive appearance that stands out without being too bold.
  • Accessibility: The component uses semantic HTML:
    • Label is in a <span> element
    • Value and icon are in a <p> element
    • Consider adding ARIA labels if the total list is interactive or part of a form
