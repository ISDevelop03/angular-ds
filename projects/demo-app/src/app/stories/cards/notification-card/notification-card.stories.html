# API Reference

Property Type Default Description
Component Inputs
title string undefined Optional title text displayed at the top of the notification card. Shown in bold text above the content. This is the primary heading for the notification (e.g., 'New Alert', 'System Update').
content string undefined Optional content text displayed below the title. Shown in smaller, medium-weight text with a line-clamp of 2 (truncates after 2 lines). Use to provide additional details or the main message of the notification (e.g., 'This notification is unread and important.', 'Your system is up to date.').
time string undefined Optional time text displayed in the top-right corner of the card. Shown in very small text. Use to indicate when the notification was received or created (e.g., '5min', '2sec', '1 hour ago'). The time is positioned absolutely in the top-right corner.
isRead boolean false When false (unread), applies distinct styling: red left border (border-l-red-500) and light background (bg-neutral-25). When true (read), uses transparent left border and transparent background. Use to visually distinguish between read and unread notifications.
link string undefined Optional URL or route path. When provided, renders an invisible anchor tag covering the entire card, making the entire card clickable for navigation. The link is accessible via screen readers using the title as the accessible text. Can be a relative path, absolute URL, or route path depending on your routing setup.
id string undefined Optional unique identifier for the notification card. Use for tracking, identification, or DOM manipulation. Currently available as an input property for future use or custom implementations.
data any undefined Optional data object passed to the onClick event handler. When the card is clicked, the handler receives an object with data and event properties: { data: any, event: Event }. Use to pass additional context or information to the click handler.
Component Outputs (Events)
onClick EventEmitter<any> - Emitted when the notification card is clicked. Provides an object with data and event properties: { data: any, event: Event }. The event is the click event, and data comes from the component's data input property. The entire card is clickable. Use to handle card selection, mark as read, navigate to details, or trigger notification actions.
