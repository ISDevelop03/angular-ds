Filial Card

# API Reference

Property Type Default Description
Component Inputs
name string - Required. The name of the filial (branch or subsidiary) displayed as the card title. Shown in bold text. The component also uses the first two letters of the name to generate an avatar/initial badge when no icon is provided.
id number - Required. Unique identifier for the filial card. Used for tracking and identification when the card is selected.
icon string undefined Optional icon identifier or image URL. When provided, displays an icon or image instead of the text initials avatar. Should match an available icon in your icon library or be a valid image URL.
sector string undefined Optional sector or category information for the filial. Can be used for filtering, grouping, or additional context. Currently available as an input property for future use or custom implementations.
useImageBorder boolean false When true, applies a border to the icon/image. Use for custom styling of the icon container. When false, uses the default styling without a border.
data any undefined Optional data object passed to the setSelected event handler. When the card is clicked, the handler receives an object with event and data properties: { event: Event, data: any }. Use to pass additional context or information to the selection handler.
className string '' Optional additional CSS classes to apply to the filial card container. Useful for custom styling or layout adjustments. The card has default hover effects (shadow, background change, border transparency).
Component Outputs (Events)
setSelected EventEmitter<any> - Emitted when the filial card is clicked. Provides an object with event and data properties: { event: Event, data: any }. The event is the click event, and data comes from the component's data input property. The entire card is clickable and has keyboard accessibility (tabindex, role). Use to handle filial selection and navigation.
