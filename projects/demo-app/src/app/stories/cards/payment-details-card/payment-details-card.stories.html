# API Reference

Property Type Default Description
Component Inputs
title string '' The title text displayed in the card header. Shown in the left column of the header row, next to the optional checkbox. This is the primary heading for the payment details (e.g., 'Article ID N째234567997656875').
titleHeader string '' Optional header text for the title column. Currently available as an input property for future use or custom implementations. Not currently displayed in the template.
date string '' Optional date string displayed in the date column. When provided, the header shows "Date" label and the content row displays the formatted date using Angular's date pipe with format "dd.MM.yyyy" (e.g., '2023-10-01' displays as '01.10.2023'). The date is shown in bold text.
price number 0 Optional price value displayed in the price column. When provided, the header shows "Montant TTC" label and the content row displays the formatted price using Angular's number pipe with format "1.2-2" and locale "fr-FR", followed by the currency. The price is shown in bold text (e.g., '1239' with currency 'USD' displays as '1 239,00 USD').
currency string '' Optional currency code displayed after the formatted price (e.g., 'USD', 'EUR', 'GBP'). Shown in the price column when price is provided. The currency appears after the formatted number.
useCheckbox boolean false When true, displays a checkbox in the header row next to the title. The checkbox state is controlled by isSelected and can be disabled via disabled. When the checkbox is toggled, the onSelect event is emitted. Use for selection functionality in lists or tables of payment details.
isSelected boolean false The checked state of the checkbox when useCheckbox is true. When true, the checkbox is checked. Use to control the selection state of the card. The checkbox state is bound to this property.
disabled boolean false When true, disables the checkbox when useCheckbox is true. The checkbox becomes non-interactive and cannot be toggled. Use to prevent selection in certain states or conditions.
data any undefined Optional data object passed to the onSelect event handler. When the checkbox is toggled, the handler receives an object with event and data properties: { event: any, data: any }. Use to pass additional context or payment information to the selection handler.
className string '' Optional additional CSS classes to apply to the payment details card container. Useful for custom styling, layout adjustments, or responsive sizing (e.g., max-w-[650px]). The card has default styling with border, background, rounded corners, and flex column layout.
Component Outputs (Events)
onSelect EventEmitter<any> - Emitted when the checkbox is toggled (only when useCheckbox is true). Provides an object with event and data properties: { event: any, data: any }. The event is the checkbox change event, and data comes from the component's data input property. Use to handle selection changes, update selection state, or perform actions based on card selection.

Additional Notes:

  • Content Projection: The component supports content projection via ng-content. You can project custom content (e.g., descriptions, additional details, HTML elements) into the main content area of the card. The projected content appears in the left column (spanning 3 columns) of the content row.
  • Layout: The card uses a 5-column grid layout:
    • Header row: Title/checkbox (3 columns), Date label (1 column), Price label (1 column)
    • Content row: Projected content (3 columns), Formatted date (1 column), Formatted price with currency (1 column)
  • Price Formatting: The price is automatically formatted using Angular's number pipe with French locale formatting (spaces as thousand separators, comma as decimal separator). The format is "1.2-2" which means: minimum 1 digit before decimal, 2 digits after decimal.
  • Date Formatting: The date is automatically formatted using Angular's date pipe with format "dd.MM.yyyy" (e.g., '2023-10-01' displays as '01.10.2023').
  • Conditional Display: The date and price columns are only displayed when their respective input properties are provided. The checkbox is only displayed when useCheckbox is true.
