Payment Card

# API Reference

Property Type Default Description
Component Inputs
title string '' Optional title text displayed in the card. Shown in semibold text next to the image. This is the primary heading for the payment card (e.g., product name, service name). The title is also used as the image alt text for accessibility.
image string '' Optional URL or path to the payment card image. The image is displayed on the left side of the card with a square aspect ratio (50px × 50px), rounded corners, and object-cover styling. The image appears between the checkbox and the title.
price number 0 Optional price value displayed in the amount section. The price is automatically formatted using the formatMoney utility function and displayed in bold text below the "Montant" label. The formatted price is shown alongside the currency.
currency string '' Optional currency code displayed after the formatted price (e.g., 'USD', 'EUR', 'GBP'). Shown in the amount section next to the formatted price value.
isSelected boolean false The checked state of the checkbox. When true, the checkbox is checked. The checkbox is always displayed on the left side of the card. When the checkbox is toggled, the onSelect event is emitted. Use to control the selection state of the payment card.
deleteText string 'Supprimer' Label text for the delete button displayed on the right side of the card. Defaults to 'Supprimer' (French for "Delete"). The delete button includes a trash icon and triggers the onDelete event when clicked. Use to customize the button text for localization or branding.
data any undefined Optional data object passed to the onSelect and onDelete event handlers. When the checkbox is toggled or the delete button is clicked, the respective handler receives an object with event and data properties: { event: any, data: any }. Use to pass additional context or payment information to the event handlers.
className string '' Optional additional CSS classes to apply to the payment card container. Useful for custom styling, layout adjustments, or responsive sizing (e.g., max-w-[800px]). The card has default styling with border, background, rounded corners, padding, and flex layout with space-between justification.
Component Outputs (Events)
onSelect EventEmitter<any> - Emitted when the checkbox is toggled. Provides an object with event and data properties: { event: any, data: any }. The event is the checkbox change event, and data comes from the component's data input property. Use to handle selection changes, update selection state, or perform actions based on card selection.
onDelete EventEmitter<any> - Emitted when the delete button is clicked. Provides an object with event and data properties: { event: Event, data: any }. The event is the click event, and data comes from the component's data input property. Use to handle deletion, remove items from lists, or trigger confirmation dialogs.

Additional Notes:

  • Layout: The card uses a horizontal flex layout with space-between justification:
    • Left side: Checkbox, image (optional), title (optional), and amount section (price + currency)
    • Right side: Delete button with trash icon
  • Checkbox: The checkbox is always displayed on the left side of the card, regardless of other content. It's used for selection functionality and cannot be hidden.
  • Amount Section: The amount section displays a "Montant" label above the formatted price and currency. The section has a light background and padding. The price is automatically formatted using the formatMoney utility function.
  • Delete Button: The delete button is always displayed on the right side of the card with a trash icon. The button text is customizable via the deleteText input property. The icon is displayed in red color.
  • Image Display: When an image is provided, it's displayed as a square (50px × 50px) with rounded corners and object-cover styling to maintain aspect ratio. The image appears between the checkbox and the title.
