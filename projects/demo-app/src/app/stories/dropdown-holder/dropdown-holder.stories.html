Dropdown Holder

This is a dropdown holder component that can be used to wrap any content. It allows for custom button content and emits an event when the dropdown is toggled.

Click the button below to toggle the dropdown. The button content can be customized using the buttonContent input.

# API Reference

Property Type Default Description
Component Inputs
buttonContent TemplateRef<any> undefined Required. Template reference for the dropdown trigger button content. Use an ng-template with a template reference variable (e.g., #buttonContent) to define custom button content. The template can contain any HTML, icons, text, or components. When provided, this content is rendered inside the trigger button.
containerClassName string '' Optional CSS classes to apply to the dropdown menu container. Use for custom styling of the dropdown panel (e.g., borders, padding, background colors, width constraints). Example: max-w-[500px] p-4 rounded-md border border-neutral-100 bg-level-3.
className string '' Optional additional CSS classes to apply to the dropdown holder wrapper. Useful for custom styling or layout adjustments of the component container.
Content Projection ng-content - The dropdown menu content uses content projection. You can project any content (HTML, components, text) into the dropdown panel. The projected content appears inside the dropdown container when the dropdown is open. Use for flexible, custom dropdown content.
Component Outputs (Events)
toggle EventEmitter<boolean> - Emitted when the dropdown is opened or closed. Provides true when the dropdown opens and false when it closes. Use to track the dropdown state and perform actions based on open/close events (e.g., updating UI, fetching data, animating icons).
