# API Reference

Property Type Default Description
Component Inputs
title string | undefined undefined Optional heading shown above the signature block. When omitted or empty, the title row is not rendered.
initiation TAction | undefined undefined First row: agent on the left, label and date on the right. Uses the same TAction shape as actions. See the TAction Interface section below.
actions TAction[] | undefined undefined Additional rows listed under the initiation row, separated by a divider and vertical connectors. Use an empty array to show only the initiation block. Each entry follows TAction; see the TAction Interface section below.
className string '' Optional extra CSS classes merged onto the root container (default includes background, rounded corners, and padding).

# TAgent Interface

The actionAgent field on TAction uses TAgent: the person's display name and role label.

Property Type Default Description
name string - Required. Signatory or agent name, shown bold on the left.
role string - Required. Role or title (e.g. 'Initiator'), shown below the name in muted text.

# TAction Interface

The initiation input and each element of actions share the TAction shape: one label/date pair on the right and an actionAgent on the left (see TAgent Interface above).

Property Type Default Description
actionDate string - Required. Date string shown bold on the right, below actionLabel.
actionLabel string - Required. Step label (e.g. 'Initiation'), muted, above the date.
actionAgent TAgent - Required. Name and role block on the left. See TAgent Interface above.
