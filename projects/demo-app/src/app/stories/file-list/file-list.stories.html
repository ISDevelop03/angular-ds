File List

Downloadable FileList with callback

# API Reference

Property Type Default Description
Component Inputs
files UploadedFiles[] [] Required. Array of file objects to display in the list. Each file object contains file information, upload status, progress, and optional error messages. The component automatically formats file sizes and displays different UI states based on the file status. See the UploadedFiles Type section below for details.
isDownloadable boolean false When true, displays a download button for each file instead of a delete button. When false, shows a delete button for successfully uploaded files. If onDownload has listeners, it emits the file; otherwise, triggers a default browser download.
isMultiple boolean false When true, indicates that multiple files can be displayed. Currently used for internal logic and future enhancements. The component supports displaying multiple files regardless of this setting.
className string '' Optional additional CSS classes to apply to the file list container. Useful for custom styling or layout adjustments.
Component Outputs (Events)
onDelete EventEmitter<string> - Emitted when a file's delete button is clicked. Provides the file ID (string) as the event value. Use to handle file deletion and remove the file from your data source. Only shown for files with status === 'success' when isDownloadable is false.
onReUpload EventEmitter<string> - Emitted when a file's re-upload button is clicked (for files with status === 'error'). Provides the file ID (string) as the event value. Use to handle file re-upload attempts and retry failed uploads.
onDownload EventEmitter<File> - Emitted when a file's download button is clicked (when isDownloadable is true). Provides the File object as the event value. If no listeners are registered, the component automatically triggers a browser download. Use to handle custom download logic (e.g., server-side downloads, custom file handling).

# UploadedFiles Type

The UploadedFiles type defines the structure for file objects passed to the files input property.

Property Type Default Description
file File & { id: string } - Required. The File object with an additional id property. The File object contains standard file properties (name, size, type, etc.). The id property must be a unique string identifier used for tracking, deletion, and error management. The component uses this ID to identify files when emitting events.
status 'uploading' | 'success' | 'error' undefined Optional upload status that determines the visual state and available actions. Available options: 'uploading' (shows progress bar), 'success' (shows success styling with delete/download button), 'error' (shows error styling with re-upload button and expandable error messages). When not provided, displays a neutral state with file information.
progress number undefined Optional upload progress percentage (0-100). Only used when status === 'uploading'. Displays a progress bar with the percentage value. The progress bar width is animated based on this value. Use to show real-time upload progress to users.
errors null | string[] undefined Optional array of error messages. Only used when status === 'error'. When provided, displays an expandable error section with a list of error messages. Users can click to expand/collapse the error list. Each error is displayed as a bulleted list item. Use to provide detailed validation or upload error information to users.
