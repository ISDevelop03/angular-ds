Sidebar

# API Reference

Property Type Default Description
Component Inputs
className string '' Optional additional CSS classes to apply to the sidebar container. Useful for custom styling, backgrounds, or layout adjustments (e.g., 'bg-level-1'). The sidebar has default styling with sticky positioning, full height, and z-index for proper layering.
profileMenus ISelectItem[] [] Array of profile menu items displayed in the bottom section of the sidebar. Each item can have a label, icon, href, and onClick handler. These menus are typically used for user profile actions, settings, or account-related options. See the ISelectItem Interface section below for details.
mainMenus IMainMenu[] [] Required. Array of main menu items displayed in the sidebar. Each menu item can have a title, icon, color, href, and optional submenu. When a menu item with submenus is clicked, it opens a sub-sidebar panel. See the IMainMenu Interface section below for details.
disableMainMenus boolean false When true, disables all main menu items, preventing clicks and interactions. Useful for read-only states or when navigation should be temporarily disabled.
disableBottomMenus boolean false When true, disables the bottom menus section (profile menus, theme toggle, language selector, etc.). This hides or disables the entire bottom section of the sidebar.
sizes Cap[] [] Array of cap size options for font size selection in the bottom menus. Each cap has a name (e.g., 'S', 'M', 'L', 'XL') and a corresponding size value. Used for accessibility features allowing users to adjust text size. See the Cap Interface section below for details.
defaultCap Cap { name: 'L', size: 18 } The default cap size selected when the sidebar loads. This determines the initial font size setting. The default is 'L' with size 18. See the Cap Interface section below for details.
logo string '/assets/logo-mybusiness.svg' URL path to the full logo image displayed in the sidebar header when the sub-sidebar is open. The logo is shown at 140px width with automatic height. Use this for the main application logo.
miniLogo string '/assets/mini-logo.svg' URL path to the mini logo image displayed in the sidebar header when the sub-sidebar is closed. The mini logo is shown at 20px width (w-5) with automatic height. Use this for a compact logo icon.
language ILanguage { src: '/assets/france-flag.svg', alt: 'Français' } Language configuration object for the language selector in the bottom menus. Contains the flag image source and alt text. See the ILanguage Interface section below for details.
subSidebarIsOpen boolean false Controls whether the sub-sidebar panel is open. When true, the sub-sidebar slides in from the right, showing the submenu items for the selected main menu. The logo switches from mini to full when the sub-sidebar is open. This should be controlled by the parent component based on the onOpenSidebar event.
subMenuData IMenus { title: '', items: [] } The menu data to display in the sub-sidebar panel. Contains a title and array of menu items. This is automatically set when a main menu item with submenus is clicked, but can also be controlled externally. See the IMenus Interface section below for details.
Component Outputs (Events)
onThemeChange EventEmitter<any> - Emitted when the theme is changed via the theme toggle in the bottom menus. The event payload contains the new theme value. Use to handle theme switching (e.g., light/dark mode) in your application.
callingOnClick EventEmitter<any> - Emitted when the calling/phone icon is clicked in the bottom menus. Use to handle calling functionality or display a calling interface.
reclamationOnClick EventEmitter<any> - Emitted when the reclamation/complaint icon is clicked in the bottom menus. Use to handle reclamation functionality or navigate to a reclamation page.
languageOnClick EventEmitter<any> - Emitted when the language selector is clicked in the bottom menus. Use to handle language switching or display a language selection menu.
settingsOnClick EventEmitter<any> - Emitted when the settings icon is clicked in the bottom menus. Use to handle settings functionality or navigate to a settings page.
onOpenSidebar EventEmitter<any> - Emitted when the sub-sidebar is opened or closed. The event payload contains: { subMenuData: IMenus, open: boolean, openMainMenuIndex: number }. Use to synchronize the sub-sidebar state with your parent component and update the subSidebarIsOpen and subMenuData inputs accordingly.

# IMainMenu Interface

The IMainMenu interface defines the structure for main menu items passed to the mainMenus input property.

Property Type Default Description
title string - Required. The title text displayed for the menu item. This is the label shown in the sidebar for this menu option.
icon string undefined Optional icon identifier displayed next to the menu title. Should match an available icon in your icon library (e.g., 'home', 'settings', 'user').
color string - Required. Color value for the menu item. Used for styling the menu item (e.g., icon color, active state color). Can be a CSS color value or theme color class.
href string - Required. URL or route path for the menu item. Used for navigation when the menu item is clicked. Can be a relative path, absolute URL, or route path depending on your routing setup.
menus IMenus undefined Optional submenu data. When provided, clicking the menu item opens a sub-sidebar panel displaying the submenu items. If not provided, clicking the menu item navigates to the href. See the IMenus Interface section below for details.

# IMenus Interface

The IMenus interface defines the structure for submenu data passed to the subMenuData input or used in the menus property of IMainMenu.

Property Type Default Description
title string - Required. The title text displayed at the top of the sub-sidebar panel. This is typically the same as the main menu item title or a category name.
items IMenuListItem[] - Required. Array of menu list items displayed in the sub-sidebar. Each item can have a title, icon, href, and optional nested items for multi-level menus. See the IMenuListItem Interface section below for details.

# IMenuListItem Interface

The IMenuListItem interface defines the structure for individual items within a submenu's items array.

Property Type Default Description
title string - Required. The title text displayed for the menu item in the sub-sidebar.
icon string undefined Optional icon identifier displayed next to the menu item title. Should match an available icon in your icon library.
href string - Required. URL or route path for the menu item. Used for navigation when the item is clicked.
isNew boolean undefined When true, indicates that this is a new menu item, which may be styled differently (e.g., with a "new" badge or highlight) to draw attention.
items IMenuListItem[] undefined Optional nested menu items for multi-level menus. Allows creating hierarchical menu structures with sub-items under a parent item.

# ILanguage Interface

The ILanguage interface defines the structure for language configuration passed to the language input property.

Property Type Default Description
src string - Required. URL path to the language flag image (e.g., '/assets/france-flag.svg'). This image is displayed in the language selector in the bottom menus.
alt string - Required. Alt text for the language flag image (e.g., 'Français'). Used for accessibility and as a fallback if the image fails to load.
className string undefined Optional additional CSS classes to apply to the language flag image. Useful for custom styling or sizing.

# Cap Interface

The Cap interface defines the structure for font size options passed to the sizes and defaultCap input properties.

Property Type Default Description
name 'S' | 'M' | 'L' | 'XL' - Required. The name of the cap size. Must be one of 'S', 'M', 'L', or 'XL'. This is displayed as the label for the font size option in the accessibility menu.
size number - Required. The font size value in pixels corresponding to the cap name. For example, 'S' might be 14px, 'M' might be 16px, 'L' might be 18px, and 'XL' might be 20px. This value is used to apply the selected font size to the application.

# ISelectItem Interface

The ISelectItem interface defines the structure for profile menu items passed to the profileMenus input property. This interface is also used in other components like profile-dropdown.

Property Type Default Description
href string undefined Optional URL or route path. When provided, the menu item is rendered as a link. Can be a relative path, absolute URL, or route path depending on your routing setup.
label string - Required. The label text displayed for the menu item.
isExternal boolean undefined When true, indicates that the link is external. This may affect how the link is rendered (e.g., opening in a new tab with target="_blank").
icon string undefined Optional icon identifier displayed next to the menu item label. Should match an available icon in your icon library.
className string undefined Optional additional CSS classes to apply to the menu item. Useful for custom styling.
onClick () => void undefined Optional callback function executed when the menu item is clicked. Use to handle custom actions that don't require navigation (e.g., opening a modal, toggling a setting).
