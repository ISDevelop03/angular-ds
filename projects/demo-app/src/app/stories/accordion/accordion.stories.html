# API Reference

Property Type Default Description
Component Inputs
variant 'default' | 'borderless' | 'borderless-reversed' | 'shadowless' 'default' Visual style variant of the accordion:
  • 'default' - Standard styling with background, shadow, and rounded corners
  • 'borderless' - Clean look without background, icon positioned on the right
  • 'borderless-reversed' - Same as borderless but with icon positioned on the left (reversed layout)
  • 'shadowless' - Similar to default but without shadow effect
items AccordionItem[] [] Array of accordion items to display. Each item represents a collapsible panel. See the AccordionItem Interface section below for details.
allowMultiple boolean false When true, allows multiple accordion panels to be open simultaneously. When false, opening one panel automatically closes all others (accordion behavior).
textClassName string '' Optional CSS classes to apply to the accordion item title text. Use for custom text styling like font size, weight, or color.
className string '' Optional additional CSS classes to apply to the accordion container. Useful for custom styling or layout adjustments.
Component Outputs (Events)
openHandler EventEmitter<AccordionItem & {{ '{' }} isOpen: boolean {{ '}' }}> - Emitted whenever a panel is toggled (open or close). The payload is the row object for that panel: all AccordionItem fields plus isOpen for the new state. Use this to sync external state or react to expand and collapse.

# AccordionItem Interface

The AccordionItem interface defines the structure for accordion items passed to the items input property.

Property Type Default Description
id string - Required. Unique identifier for the accordion item. Used internally to track which panel is open/closed.
title string - Required. The text displayed in the accordion header button. This is the visible label for the collapsible panel.
content string | TemplateRef<any> - Required. Body shown when the panel is open. Use a string for HTML (bound with innerHTML) or a TemplateRef for structured markup. For a TemplateRef, the outlet context is the item’s context property (pass an object your template can read with let-* bindings).
caption string undefined Optional secondary text displayed below or alongside the title. Use for additional context or descriptions.
buttonTemplate string | TemplateRef<any> undefined Optional extra controls in the header row, after the title and chevron (for example an action button). String or TemplateRef. For a template, the outlet context exposes the row as the template variable item (all AccordionItem fields plus isOpen). Use $event.stopPropagation() on inner controls if they should not toggle the panel.
prefixTemplate string | TemplateRef<any> undefined Optional block rendered before the title area (for example a checkbox or icon). Same template context as buttonTemplate: same item template variable as the row. Prefer stopPropagation on interactive elements so the header button does not receive the click.
context any undefined Data passed only to the content TemplateRef outlet (not to buttonTemplate or prefixTemplate). Set this when content is a template so the panel body can read your model.
count string | number undefined Optional badge count displayed next to the title. Can be a number or string (e.g., '5' or '+9'). When provided, displays as a circular badge with the specified countColor.
countColor string undefined Optional CSS class for the count badge background color. Defaults to 'bg-red-500' if not specified. Use Tailwind color classes like 'bg-yellow-500', 'bg-purple-500', etc.
open boolean undefined Optional initial open state for the accordion panel. When true, the panel starts expanded. When allowMultiple is false, only one panel can be open at a time, so setting multiple items to open: true will result in only the last one being open.
