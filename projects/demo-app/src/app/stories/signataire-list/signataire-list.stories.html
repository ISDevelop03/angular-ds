# API Reference

Property Type Default Description
Component Inputs
items SignataireItem[] [] Required. Array of signataire items to display in the list. Each item represents a signer with their name and signing date. Items are displayed vertically with a divider between each entry. The left side shows the label and value (typically the signer's name), while the right side shows the date label and date value. See the SignataireItem Interface section below for details.
className string '' Optional additional CSS classes to apply to the container element. Useful for custom styling, layout adjustments, or responsive sizing (e.g., max-w-[400px]). The component has default styling with a neutral background, padding, rounded corners, and vertical dividers between items.

# SignataireItem Interface

The SignataireItem interface defines the structure for signataire items passed to the items input property.

Property Type Default Description
label string - Required. The label text displayed above the value on the left side. Typically used for field names like "Nom" (Name), "Signataire" (Signer), etc. Displayed in a smaller, medium-weight font with neutral gray color (text-neutral-500).
value string - Required. The main value displayed below the label on the left side. Typically used for the signer's name or identifier. Displayed in a bold font with dark color (text-neutral-900) to emphasize the primary information.
dateLabel string - Required. The label text displayed above the date value on the right side. Typically used for date field names like "Date de signature" (Signing Date), "Date", etc. Displayed in a smaller, medium-weight font with neutral gray color (text-neutral-500), right-aligned.
dateValue string - Required. The date value displayed below the date label on the right side. Can be any date format string (e.g., "2025-01-01", "01/01/2025", "January 1, 2025"). Displayed in regular weight with dark color (text-neutral-900), right-aligned.
