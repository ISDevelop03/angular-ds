# API Reference

Property Type Default Description
Component Inputs
value string '' The current value of the textarea. Supports two-way data binding with [(value)]. The value is bound to the textarea's value attribute and updates when the user types.
label string undefined Optional label text displayed above the textarea. The label is only rendered if provided (conditional rendering with *ngIf). If required is true, a red asterisk (*) is displayed next to the label. The label can also include a tooltip icon if tooltip is provided.
placeholder string undefined Optional placeholder text displayed in the textarea when it's empty. Provides a hint to the user about what to enter. The placeholder text is styled with gray color and larger font size.
name string undefined Optional name attribute for the textarea element. Useful for form submission and identification. If not provided, the textarea won't have a name attribute.
rows number 1 The number of visible text lines (rows) for the textarea. Controls the initial height of the textarea. The textarea can grow beyond this if resize is true or if the content exceeds the visible area. Common values: 3, 4, 5, or more depending on the expected content length.
maxLength number undefined Optional maximum number of characters allowed in the textarea. When provided, the textarea's maxlength attribute is set, preventing users from entering more characters than specified. Use for fields with character limits (e.g., comments, descriptions, messages).
resize boolean true When true, allows the textarea to be resized by the user (default browser behavior). When false, the textarea has a fixed size and cannot be resized (resize-none CSS class is applied). Use false for consistent layouts where you don't want users to change the textarea size.
disabled boolean false When true, disables the textarea, preventing user input. Disabled textareas have:
  • Gray background: bg-gray-50
  • Not-allowed cursor: cursor-not-allowed
  • No user interaction possible
Use for read-only states or when the field should be temporarily unavailable.
required boolean false When true, indicates that the textarea is required. This displays a red asterisk (*) next to the label. Use for form validation and to indicate mandatory fields to users. Note: This is a visual indicator only; actual validation should be handled in your form logic.
hasError boolean false When true, indicates that the textarea has a validation error. Error state styling includes:
  • Red border: border-error-400 (default), border-error-500 (hover/focus)
  • Error ring on focus: focus-within:ring-error-500
Use in combination with errorMessage to display validation feedback to users.
errorMessage string undefined Optional error message text displayed below the textarea when hasError is true. The error message is styled in red (text-error-500) and appears below the textarea. The message is only rendered if both hasError and errorMessage are provided (conditional rendering with *ngIf).
description string undefined Optional description text displayed below the textarea. Provides additional context, hints, or instructions to users (e.g., "Minimum 100 characters", "Optional field", "Enter your message here"). The description is styled in gray (text-gray-500) and appears below the textarea. The description is only rendered if provided (conditional rendering with *ngIf).
tooltip string | TemplateRef<any> undefined Optional tooltip content displayed when hovering over an info icon next to the label. Can be either:
  • String: Simple text tooltip
  • TemplateRef: Complex HTML content tooltip
The tooltip icon (info-circle) is only displayed if tooltip is provided. The tooltip appears to the right of the label using the ds-tooltip component.
variant keyof typeof textarea 'default' Theme variant for styling the textarea. Currently only 'default' is available. The variant determines the visual styling, colors, and appearance of the textarea, label, error message, and description. Future variants may be added for different color schemes or styles.
Component Outputs (Events)
valueChange EventEmitter<string> - Emitted when the textarea value changes (user types or pastes text). The event payload is the new string value of the textarea. This event is used for two-way data binding with [(value)]. Use to handle value changes, perform validation, or update parent component state.
