# API Reference

Property Type Default Description
Component Inputs
items SelectItem[] [] Required. Array of selectable options. Each item defines an option with its value, label, and optional properties. The options are displayed in the dropdown menu. See the SelectItem Interface section below for details.
value string | string[] | null null The currently selected value(s). For single select, this is a string or null. For multiple select (multiple: true), this is an array of strings. Supports two-way data binding with [(value)]. The component automatically finds and displays the corresponding item(s) from the items array.
label string undefined Optional label text displayed above the select button. The label is only rendered if provided (conditional rendering with *ngIf). If required is true, a red asterisk (*) is displayed next to the label. The label can also include a tooltip icon if tooltip is provided.
placeholder string 'Select an option' Placeholder text displayed in the select button when no option is selected. Provides a hint to the user about what to select. The placeholder is styled differently from selected values (semibold, neutral-900 color).
multiple boolean false When true, enables multiple selection mode. Users can select multiple options, and the value becomes an array of strings. In multiple mode, the dropdown doesn't close after selecting an option, and selected items are highlighted. The selected label displays all selected item labels joined with commas.
searchable boolean false When true, shows a search/filter input inside the dropdown. Users can type to filter options by label (case-insensitive). Useful when the options list is long. Unlike autoComplete, the select button still shows the selected label (or placeholder); only the dropdown content includes the search field.
autoComplete boolean false When true, enables autocomplete/search functionality. The select button displays an input field instead of the selected label, allowing users to type and filter options. The options are filtered based on the search term matching item labels (case-insensitive). A clear button (close icon) appears when a value is selected. The onSearch event is emitted when the user types.
debounceTime number 300 The debounce time in milliseconds for search input when autoComplete is true. This delays the onSearch event emission to reduce the number of events fired while the user is typing. Higher values reduce event frequency but increase perceived delay.
disabled boolean false When true, disables the select, preventing user interaction. Disabled selects have:
  • Gray background: bg-neutral-100
  • Gray text: text-neutral-500
  • Not-allowed cursor: cursor-not-allowed
  • No hover effects
  • Dropdown cannot be opened
Use for read-only states or when the field should be temporarily unavailable.
required boolean false When true, indicates that the select is required. This displays a red asterisk (*) next to the label. Use for form validation and to indicate mandatory fields to users. Note: This is a visual indicator only; actual validation should be handled in your form logic.
allowClear boolean false When true, displays a clear (X) button when a value is selected, allowing users to clear their selection.
hasError boolean false When true, indicates that the select has a validation error. Error state styling includes:
  • Red border and ring: ring-error-400 border-error-400
  • Error message displayed below (if provided)
Use in combination with errorMessage to display validation feedback to users.
errorMessage string undefined Optional error message text displayed below the select when hasError is true. The error message is styled in red (text-error-500) and appears below the select button. The message is only rendered if both hasError and errorMessage are provided (conditional rendering with *ngIf).
description string undefined Optional description text displayed below the select. Provides additional context, hints, or instructions to users. The description is styled in gray (text-gray-500) and appears below the select button. The description is only rendered if provided (conditional rendering with *ngIf).
tooltip string | TemplateRef<any> undefined Optional tooltip content displayed when hovering over an info icon next to the label. Can be either:
  • String: Simple text tooltip
  • TemplateRef: Complex HTML content tooltip
The tooltip icon (info-circle) is only displayed if tooltip is provided. The tooltip appears to the right of the label using the ds-tooltip component.
isLoading boolean false When true, displays a loading spinner icon instead of the arrow icon. The spinner is animated (animate-spin) and indicates that options are being loaded. The dropdown cannot be opened while loading. Use for async data loading scenarios.
emptyLabel string 'No options' Text displayed in the dropdown when there are no options available (empty items array or all items filtered out). This provides feedback to users when the select has no selectable options.
pill boolean false When true, applies rounded-full styling to the select button, giving it a pill-shaped appearance. Use for a more modern or compact look.
buttonIcon string undefined Optional icon identifier displayed on the left side of the select button. Should match an available icon in your icon library (e.g., 'message', 'user', 'settings'). The icon is displayed in red color (text-red-500) when enabled, or gray when disabled. If a selected item has an icon property, it takes precedence over buttonIcon.
variant 'default' | 'currency' 'default' Theme variant for styling the select component. Available options:
  • 'default': Standard select styling with border, background, and standard positioning
  • 'currency': Currency-specific styling with transparent background, compact sizing, and right-aligned options dropdown
Each variant has different visual styling for the button, options container, and layout.
customOptionsStyles {{ '{' }} [key: string]: any {{ '}' }} undefined Optional inline styles object to apply to the options dropdown container. Each key-value pair is applied as a CSS property-value pair. Useful for custom positioning, width, height, or other specific styling that can't be achieved with CSS classes alone. Styles are merged with default positioning styles.
className string undefined Optional additional CSS classes to apply to the select container. Useful for custom styling, spacing, or layout adjustments. The component has default styling with relative positioning and full width.
Component Outputs (Events)
valueChange EventEmitter<any> - Emitted when an option is selected or deselected. The event payload is an object with {{ '{' }} value: string | string[], event: Event {{ '}' }}. For single select, value is a string. For multiple select, value is an array of strings. This event is used for two-way data binding with [(value)]. Use to handle selection changes, perform validation, or update parent component state.
onSearch EventEmitter<string> - Emitted when the user types in the autocomplete input (only when autoComplete is true). The event payload is the search term string. The event is debounced based on debounceTime. Use to perform remote searches, filter options dynamically, or handle search input for custom filtering logic.

# SelectItem Interface

The SelectItem interface defines the structure for option items passed to the items input property.

Property Type Required Description
value string Yes The unique value identifier for the option. This is the value that will be stored in the value input when the option is selected. Must be unique within the items array.
label string Yes The display text for the option. This is what users see in the dropdown menu and in the select button when the option is selected. For account-type items (isAccount: true), this is displayed as a secondary label above the value.
unavailable boolean No When true, marks the option as unavailable. Unavailable options:
  • Have reduced opacity (opacity-50)
  • Show not-allowed cursor (cursor-not-allowed)
  • Cannot be selected (click handler returns early)
  • Don't have hover effects
Use for options that are temporarily unavailable or disabled.
icon string No Optional icon identifier displayed next to the option label. Should match an available icon in your icon library. For regular items, the icon appears on the left side. For account-type items (isAccount: true), the icon is displayed in a circular container (40px × 40px) with red color. If provided, the icon is also used in the select button when the item is selected.
image string No Optional URL path to an image displayed next to the option label. For regular items, the image is displayed as a circular 20px × 20px image. For account-type items (isAccount: true), the image is displayed as a circular 40px × 40px image. If provided, the image is also used in the select button when the item is selected (takes precedence over icon). Use for avatars, logos, or thumbnails.
isAccount boolean No When true, renders the option in account-style layout:
  • Larger icon/image (40px × 40px instead of 20px × 20px)
  • Two-line layout: label as secondary text (smaller, gray) above, value as primary text (bold) below
  • Left border indicator when selected (red bar)
Use for account selection or similar two-line option displays.
iconClass string No Optional CSS classes to apply to the icon element. Useful for custom icon styling, colors, or sizes. Applied directly to the ds-icon component when the icon is rendered.
imageClass string No Optional CSS classes to apply to the image element. Useful for custom image styling, borders, or effects. Applied directly to the <img> element when the image is rendered.
