# API Reference

Property Type Default Description
Component Inputs
checked boolean false The checked state of the checkbox. When true, the checkbox is checked and displays a check icon. When false, it is unchecked. Supports two-way binding.
isIndeterminate boolean false When true, displays a minus icon instead of a check icon, indicating an indeterminate state (partially selected). Use for scenarios where some but not all items in a group are selected. When false, uses the normal check icon when checked.
label string undefined Optional text label displayed next to the checkbox. When provided, appears to the right of the checkbox input. The label styling changes based on the checked and disabled states.
disabled boolean false When true, disables the checkbox, preventing user interaction. The checkbox appears grayed out with neutral colors and cannot be toggled. The label also appears in a disabled style.
hasError boolean false When true, applies error styling to the checkbox (red border). Use to indicate validation errors. The error message is displayed when both hasError and errorMessage are provided.
errorMessage string undefined Optional error message text displayed below the checkbox when hasError is true. The message appears in red text. Use to provide validation feedback to users.
variant keyof typeof checkbox 'default' Visual style variant of the checkbox. Available options: 'default', 'autocomplete'. Each variant has distinct wrapper, checkbox, and label styling. The 'autocomplete' variant is optimized for use within autocomplete components with smaller text and different spacing.
name string undefined Optional HTML name attribute for the checkbox input. Use for form submission and grouping checkboxes. When multiple checkboxes share the same name, they can be grouped together.
value string undefined Optional HTML value attribute for the checkbox input. Use for form submission to identify which checkbox was selected when multiple checkboxes share the same name.
change () => void () => {} Callback function executed when the checkbox state changes. Use to handle checkbox toggle events and update the checked state or perform other actions when the checkbox is clicked.
className string undefined Optional additional CSS classes to apply to the checkbox wrapper. Useful for custom styling or layout adjustments.
