# API Reference

Property Type Default Description
Component Inputs
id string - Required. Unique identifier for the input element. Used to associate the label with the input for accessibility. Also used as the for attribute on the label element.
name string - Required. Name attribute for the input element. Used for form submission and Angular forms integration. Should match the form control name when using reactive forms.
label string undefined Optional label text displayed above the input field. When provided, a label element is rendered and associated with the input via the id attribute. The label is hidden when type is 'hidden'. A red asterisk (*) is automatically appended when required is true.
variant 'default' | 'search' 'default' Visual style variant for the input component. 'default': standard input styling with neutral background. 'search': search input styling with different background and label styling. Each variant has its own theme configuration for wrapper, label, input, and error states.
display 'inline' | 'block' 'block' Layout display mode for the input and label. 'block': label appears above the input (default). 'inline': label and input are displayed side-by-side in a flex row. When inline, the label margin is adjusted and positioned at the top if description or error message is present.
type string 'text' HTML input type attribute (e.g., 'text', 'email', 'password', 'number', 'tel', 'url', 'hidden'). When set to 'hidden', the label is not rendered and the wrapper border is removed. When set to 'number', special styling is applied to hide number input spinners. Standard HTML input types are supported.
placeholder string undefined Optional placeholder text displayed inside the input when it's empty. Provides hints or examples of expected input format. The placeholder has different styling in light and dark modes.
prefix string undefined Optional text or content displayed before the input field (on the left side). Can be a string or projected content via ng-content select="[prefix]". If both string and projected content are provided, projected content takes precedence. Useful for currency symbols, icons, or other prefixes.
suffix string undefined Optional text or content displayed after the input field (on the right side). Can be a string or projected content via ng-content select="[suffix]". If both string and projected content are provided, projected content takes precedence. Useful for units, icons, or action buttons (e.g., password toggle).
value any undefined The input value. Can be bound using two-way binding with [(value)] or one-way binding with [value]. The component implements ControlValueAccessor, so it can be used with Angular reactive forms and template-driven forms. Value changes are automatically synchronized with Angular forms.
disabled boolean false When true, disables the input field. The input becomes non-interactive, shows disabled styling (gray background, reduced opacity), and prevents user input. Also sets the disabled state for Angular forms integration.
required boolean false When true, marks the input as required. A red asterisk (*) is automatically appended to the label text. Use with Angular forms validation to indicate mandatory fields. The HTML required attribute is also set on the input element.
hasError boolean false When true, applies error styling to the input (red border, red text). Should be used together with errorMessage to display validation errors. The error state overrides hover and focus states with red colors.
errorMessage string undefined Optional error message text displayed below the input field. Shown in red text when hasError is true. Use to display validation error messages from Angular forms or custom validation logic. The label text also turns red when both hasError and errorMessage are provided.
description string undefined Optional descriptive text displayed below the input field. Shown in smaller gray text. Use to provide additional context, hints, or instructions about the expected input. Displayed above the error message if both are present.
tooltip string | TemplateRef<any> undefined Optional tooltip content displayed next to the label. Can be a string or a template reference. When provided, an info icon is shown next to the label, and hovering over it displays the tooltip. Useful for providing additional help or context about the input field.
password boolean false When true, enables password visibility toggle functionality. The input type automatically switches between 'password' and 'text' based on the toggle state. A toggle button is typically shown in the suffix area to allow users to show/hide the password. Use for password input fields.
className string '' Optional additional CSS classes to apply to the input component container. Useful for custom styling, layout adjustments, or spacing. Applied to the outermost wrapper element.
inputClassName string '' Optional additional CSS classes to apply directly to the input element. Useful for custom input styling that doesn't affect the wrapper or other elements. Applied to the native <input> element.
Component Outputs (Events)
onClick EventEmitter<Event> - Emitted when the input field is clicked. Provides the native click event. Use to handle click interactions on the input field.
onInput EventEmitter<Event> - Emitted on every input change (as the user types). Provides the native input event. The component automatically updates its internal value and notifies Angular forms. Use to handle real-time input changes, validation, or live updates.
onChange EventEmitter<Event> - Emitted when the input value changes and the field loses focus (blur). Provides the native change event. Use to handle value changes that occur when the user finishes editing the field.
onFocus EventEmitter<FocusEvent> - Emitted when the input field receives focus. Provides the native focus event. Use to handle focus interactions, show additional UI, or trigger validation.
onBlur EventEmitter<FocusEvent> - Emitted when the input field loses focus. Provides the native blur event. The component automatically marks the field as touched for Angular forms. Use to handle blur interactions, trigger validation, or save data.
onKeydown EventEmitter<KeyboardEvent> - Emitted when a key is pressed down while the input has focus. Provides the native keyboard event. Use to handle keyboard interactions, shortcuts, or special key handling.
onKeyup EventEmitter<KeyboardEvent> - Emitted when a key is released while the input has focus. Provides the native keyboard event. Use to handle keyboard interactions after a key is released.
onEnter EventEmitter<KeyboardEvent> - Emitted when the Enter key is pressed while the input has focus. Provides the keyboard event. Use to submit forms, trigger actions, or navigate to the next field.
onEscape EventEmitter<KeyboardEvent> - Emitted when the Escape key is pressed while the input has focus. Provides the keyboard event. Use to cancel actions, clear the input, or close modals.
onPaste EventEmitter<ClipboardEvent> - Emitted when content is pasted into the input field. Provides the clipboard event. Use to handle paste operations, validate pasted content, or transform pasted data.
onSelect EventEmitter<UIEvent> - Emitted when text is selected in the input field. Provides the UI event. Use to handle text selection, show context menus, or provide selection-based actions.
