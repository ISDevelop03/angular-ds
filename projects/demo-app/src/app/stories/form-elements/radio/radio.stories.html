Radio

Default Variant

# API Reference

Property Type Default Description
Component Inputs
modelValue string undefined The currently selected value for the radio group. Supports two-way data binding with [(modelValue)]. The radio button is checked when modelValue equals value. All radio buttons in the same group should share the same modelValue binding and have the same name attribute.
value string undefined Required. The unique value for this radio button. When this radio is selected, modelValue is set to this value. Each radio button in a group must have a unique value. The radio button is checked when modelValue === value.
name string undefined Required for radio groups. The name attribute for the radio button. All radio buttons in the same group must have the same name value. This ensures that only one radio button in the group can be selected at a time. Also used for form submission and accessibility.
label string undefined Optional label text displayed next to the radio button. The label is only rendered if provided (conditional rendering with *ngIf). The label is clickable and will select the radio button when clicked. The label styling changes based on the radio's state (checked, disabled, error).
disabled boolean false When true, disables the radio button, preventing user interaction. Disabled radio buttons have:
  • Reduced opacity (opacity-50)
  • Not-allowed cursor (cursor-not-allowed)
  • Gray background and border
  • Cannot be clicked or selected
Use for read-only states or when the option should be temporarily unavailable.
hasError boolean false When true, indicates that the radio button has a validation error. Error state styling includes:
  • Red border: border-error-500
  • Error message displayed below (if provided)
  • Error label styling (red text)
Use in combination with errorMessage to display validation feedback to users.
errorMessage string undefined Optional error message text displayed below the radio button when hasError is true. The error message is styled in red (text-error-500) and appears below the radio button. The message is only rendered if both hasError and errorMessage are provided (conditional rendering with *ngIf).
variant keyof typeof radio 'default' Theme variant for styling the radio button. Currently only 'default' is available. The variant determines the visual styling, colors, and appearance of the radio button, label, and error message. Future variants may be added for different color schemes or styles.
className string undefined Optional additional CSS classes to apply to the radio button wrapper. Useful for custom styling, spacing, or layout adjustments. The classes are merged with the default wrapper classes from the theme.
Component Outputs (Events)
modelValueChange EventEmitter<string> - Emitted when the radio button is selected. The event payload is the value of the selected radio button. This event is used for two-way data binding with [(modelValue)]. The event is only emitted if the radio button is not disabled. Use to handle selection changes, perform validation, or update parent component state.
