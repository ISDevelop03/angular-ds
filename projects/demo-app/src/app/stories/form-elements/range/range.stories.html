# API Reference

Property Type Default Description
Component Inputs
rangeType 'single' | 'range' 'single' Determines the type of range slider:
  • 'single': Single-thumb slider for selecting one value. Uses value and valueChange.
  • 'range': Two-thumb slider for selecting a range (min/max). Uses values and valuesChange.
The component renders different markup and handles different events based on this property.
min number 0 The minimum value of the range. Users cannot select a value below this. For range type, this is the minimum for both thumbs. The value is clamped to this minimum if it goes below.
max number 100 The maximum value of the range. Users cannot select a value above this. For range type, this is the maximum for both thumbs. The value is clamped to this maximum if it goes above.
step number 1 The step increment for the range slider. Values are snapped to the nearest step when dragging or clicking. For example, if step is 10, values will be 0, 10, 20, 30, etc. Use decimal values (e.g., 0.1, 0.5) for finer control, or larger values (e.g., 100, 1000) for coarser control.
unit string '' Optional unit text displayed after the value in the tooltip. For example, if unit is "€", the tooltip will show "100 €". Use for currency symbols, measurement units (e.g., "kg", "km"), or other value qualifiers.
value number min The current value for single-thumb mode (rangeType: 'single'). Supports two-way data binding with [(value)]. The value is automatically clamped between min and max and snapped to the nearest step. This property is only used when rangeType is 'single'.
values [number, number] [min, max] The current range values for two-thumb mode (rangeType: 'range'). A tuple of two numbers: [minValue, maxValue]. Supports two-way data binding with [(values)]. The values are automatically clamped between min and max, snapped to the nearest step, and constrained so that minValue <= maxValue. This property is only used when rangeType is 'range'.
variant RangeVariant 'default' Theme variant for styling the range slider. Currently only 'default' is available. The variant determines the visual styling, colors, and appearance of the track, range fill, and thumb. Future variants may be added for different color schemes or styles.
Component Outputs (Events)
valueChange EventEmitter<number> - Emitted when the value changes in single-thumb mode (rangeType: 'single'). The event payload is the new number value. This event is used for two-way data binding with [(value)]. The event is emitted continuously while dragging and when the value is set programmatically. Use to handle value changes, perform validation, or update parent component state.
valuesChange EventEmitter<[number, number]> - Emitted when the range values change in two-thumb mode (rangeType: 'range'). The event payload is a tuple of two numbers: [minValue, maxValue]. This event is used for two-way data binding with [(values)]. The event is emitted continuously while dragging either thumb and when values are set programmatically. Use to handle range changes, perform validation, or update parent component state.
