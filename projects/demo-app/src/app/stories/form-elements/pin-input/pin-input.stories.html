Pin Input

# API Reference

Property Type Default Description
Component Inputs
codeLength number 4 Number of input fields in the PIN input. Determines how many individual input boxes are displayed. Common values are 4, 6, or 8 for PIN codes, OTP codes, or verification codes.
code number | string '' The current PIN code value. Can be a number or string. Use to set an initial value or bind the code value for two-way data binding. The code is distributed across the individual input fields.
inputType 'text' | 'tel' | 'password' 'tel' HTML input type for each input field. 'tel': Optimized for numeric input on mobile devices (shows numeric keypad). 'text': Standard text input. 'password': Masks input characters. Use 'tel' for numeric PINs, 'text' for alphanumeric codes, or 'password' for hidden input.
isCodeHidden boolean false When true, hides the entered code characters (similar to password input). Use for sensitive PIN codes where characters should not be visible. When false, entered characters are visible.
isCharsCode boolean false When true, allows alphanumeric characters in addition to numbers. When false, only numeric input is allowed. Use for alphanumeric verification codes or PINs that can contain letters.
disabled boolean false When true, disables all input fields. The PIN input becomes non-interactive and users cannot enter or modify the code. Use to prevent input in certain states or during processing.
initialFocusField number 0 Index of the input field that should receive focus initially (0-based). When the component loads, this field is automatically focused. Use to control which field is focused on mount (e.g., 0 for first field, 1 for second field).
isPrevFocusableAfterClearing boolean false When true, allows focusing the previous input field when the current field is cleared (e.g., when Backspace is pressed on an empty field). When false, clearing a field does not move focus backward. Use to control navigation behavior when users delete characters.
isFocusingOnLastByClickIfFilled boolean false When true, clicking anywhere on the PIN input when all fields are filled will focus the last (rightmost) input field. When false, clicking behavior follows standard focus rules. Use to provide quick access to the last field for editing.
autocapitalize string '' HTML autocapitalize attribute value for the input fields. Controls automatic capitalization behavior on mobile devices. Common values: 'none', 'sentences', 'words', 'characters'. Use 'none' or empty string for PIN codes to prevent unwanted capitalization.
className string '' Optional additional CSS classes to apply to the PIN input wrapper. Useful for custom styling, layout adjustments, or responsive sizing. The component has default styling with full width, large text size, and maximum width constraint.
Component Outputs (Events)
onCodeChanged EventEmitter<any> - Emitted whenever the code value changes (on every character input or deletion). Provides the current code value. Use to track code changes in real-time, validate partial codes, or update UI based on input progress. This event fires for every character entered or removed.
onCodeCompleted EventEmitter<any> - Emitted when all input fields are filled and the code is complete. Provides the complete code value. Use to trigger validation, submission, or automatic processing when the user has entered the full PIN or verification code. This event fires once when the last character is entered.

Additional Notes:

  • Web Component: This component is a wrapper around the code-input web component. All inputs are passed through to the underlying web component, and events are forwarded from the web component to Angular event emitters.
  • Auto-focus and Navigation: The component automatically manages focus between input fields:
    • When a character is entered, focus moves to the next field automatically
    • When Backspace is pressed on an empty field, focus may move to the previous field (if isPrevFocusableAfterClearing is enabled)
    • Initial focus is set on the field specified by initialFocusField
  • Styling: The component uses CSS custom properties (variables) for styling the input fields:
    • Item spacing, height, borders, and border radius
    • Border colors for default, focused, and filled states
    • Background colors and font weights
    The default styling provides a clean, modern appearance with red focus borders.
  • Mobile Optimization: When inputType is set to 'tel', mobile devices will show a numeric keypad, making it easier for users to enter PIN codes on touch devices.
  • Paste Support: The underlying web component supports pasting codes from the clipboard. When a code is pasted, it's automatically distributed across the input fields.
