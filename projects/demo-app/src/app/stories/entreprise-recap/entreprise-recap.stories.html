# API Reference

Property Type Default Description
Component Inputs
items TEntrepriseRecapItems[] [] Rows to display. Each element follows TEntrepriseRecapItems. Scalar values render on one line; account-shaped values use the layout described in Account value object below. See the TEntrepriseRecapItems Interface section for the full shape.
className string '' Optional extra CSS classes merged onto the recap container (default layout includes background, flex column, gap, rounded corners, and padding).

TEntrepriseRecapItems Interface

Each object in the items array matches TEntrepriseRecapItems: a row label plus a value that is either a scalar or an account object (see Account value object).

Property Type Default Description
label string - Required. Left-aligned caption for the row (e.g. 'Entreprise', 'Compte émetteur').
value string | number | TAccount - Required. Shown on the right. Strings and numbers render as a single bold line. Objects with account_type and account_number render as two lines (type bold, number muted). See TAccount below.

#TAccount

When value is not a string or number, use this shape so the component can show account type and number on separate lines.

Property Type Default Description
account_type string - Required. Account category label (e.g. 'Compte courant'), shown bold and right-aligned.
account_number string - Required. Full account number, shown below the type in muted text.
