Tables

The table component displays tabular data with support for sorting, row selection (single or multi), expandable rows, custom cell templates, and optional expandable/truncatable column content. Use the variant input for different visual styles (default, secondary).

Default Table

Basic table with data and columns. No selection or sorting.

Selection Enabled Table (Multi)

Row selection with checkboxes; selectionMode="multi". Use header checkbox to select all. Listen to selectionChange for selected rows.

Selection Enabled Table (Single)

Only one row can be selected at a time; selectionMode="single". selectionChange emits the selected row object.

Secondary Variant Table

Use variant="secondary" for rounded borders and neutral background. Optionally set className for container height (e.g. h-[300px]).

Expandable Rows Table

Click the + / − in each row to expand or collapse the row details.

# API Reference

Inputs, outputs, and column options for the ds-table component.

Property Type Default Description
Component Inputs
data any[] []

Array of data objects to display in the table. Each object represents a row, and properties should match the accessor values in column definitions.

Selection-related properties: When selectionEnabled is true, each data object can optionally include:

  • isSelectable (boolean): When set to false, the row's checkbox is disabled and cannot be selected. Defaults to true if not specified.
  • isSelected (boolean): When set to true, the row is pre-selected on initialization. Only one row with isSelected: true will be selected initially.

Note: The data array is copied internally to ensure immutability. Any changes to the original array will not affect the table's display.

columns ColumnDef[] [] Array of column definitions that specify how each column should be rendered. See the ColumnDef Interface section below for details.
variant 'default' | 'secondary' | 'bordered' 'default' Visual style variant of the table. 'default' for standard styling, 'secondary' for rounded borders with neutral background, 'bordered' for full border styling.
remoteSort boolean false When true, sorting is handled server-side. The sortChange event will be emitted but the table won't sort data locally. Use this for large datasets or when sorting logic is on the backend.
selectionEnabled boolean false

Enables row selection with checkboxes. When true, adds a checkbox column and header checkbox for select-all functionality. Use for bulk actions or multi-row operations.

Note: See the data property documentation above for details on using isSelectable and isSelected properties in data objects.

selectionMode 'single' | 'multi' 'multi'

Determines the selection behavior when selectionEnabled is true.

  • 'multi': Allows selecting multiple rows simultaneously using checkboxes. The header checkbox enables select-all functionality.
  • 'single': Only one row can be selected at a time. Selecting a new row automatically deselects the previously selected row.

Note: This property is only effective when selectionEnabled is set to true.

className string undefined Optional additional CSS classes to apply to the table container. Useful for custom styling or layout adjustments.
expandableRows boolean false When true, each row shows an expand/collapse control (arrow) in the last column. Clicking it toggles a detail row below. Use with expandedRowTemplate to customize the expanded content.
expandedRowTemplate TemplateRef<{ $implicit: any; row: any }> undefined Optional template for the expanded row content when expandableRows is true. The template receives row (the row data). If not provided, expanded content shows the row object as JSON.
Component Outputs (Events)
sortChange EventEmitter<{ accessor: string | null; direction?: 'asc' | 'desc' }> - Emitted when a column header is clicked for sorting. Provides the column accessor and sort direction ('asc' or 'desc'). Use with remoteSort for server-side sorting or to track sort state.
selectionChange EventEmitter<any[]> - Emitted when row selection changes (only when selectionEnabled is true). Returns an array of selected row objects. Use for bulk operations like delete, export, or batch processing.

# ColumnDef Interface

The ColumnDef interface defines the structure for column configurations passed to the columns input property.

Property Type Default Description
header string - Required. The text displayed in the column header. This is the visible label for the column.
accessor string - Required. The property key used to access data from each row object. Must match a property name in your data objects (e.g., 'name', 'age', 'id').
sortable boolean false When true, enables sorting for this column. Adds sort indicators (arrows) to the header and makes the header clickable. Use for columns that should be sortable by users.
expandable boolean false When true, allows column content to be expanded/collapsed. Long text is truncated by default and can be expanded to show full content. Automatically sets truncate to true. Use for columns with potentially long content like descriptions or notes.
truncate boolean false When true, truncates multi-line text to show only the first line. Automatically set to true when expandable is true. Use to keep table rows compact when displaying long text content.
style { [key: string]: string } undefined Optional inline styles object applied to the column header and cells. Keys are CSS property names (e.g. width, textAlign). Use for column-specific styling (e.g. { width: '1%' } for auto-sizing).
cellTemplate TemplateRef<any> undefined Optional Angular template reference for custom cell rendering. The template receives the cell value and the full row object. Use for custom formatting, badges, buttons, or any complex cell content that can't be achieved with plain text.
