# API Reference

Property Type Default Description
Component Inputs
isShown boolean false Controls the visibility of the popup. When true, the popup is displayed with backdrop and animations. When false, the popup is hidden (removed from DOM via *ngIf). Use to show/hide the popup programmatically.
variant 'modal' | 'slideLeft' | 'slideRight' 'modal' Visual style and positioning variant for the popup:
  • 'modal': Centered modal dialog with fade-in and scale animation. Positioned in the center of the screen with backdrop.
  • 'slideLeft': Slides in from the right side of the screen. Fixed to the right edge, full height. Uses slide-in-left animation.
  • 'slideRight': Slides in from the left side of the screen. Fixed to the left edge, full height. Uses slide-in-right animation.
Each variant has its own positioning, styling, and animation.
heading string undefined Optional heading text displayed at the top of the popup panel. Shown in bold text above the content. When provided, a heading element is rendered. Use to provide a title or label for the popup (e.g., 'Modal Popup', 'Slide Left Popup').
close () => void () => {} Callback function executed when the popup is closed. Called when:
  • The close button (X icon) is clicked
  • The Escape key is pressed
  • The backdrop is clicked (outside the popup panel)
Use to update state, clean up resources, or perform actions when the popup closes. The component also sets isShown to false internally.
className string '' Optional additional CSS classes to apply to the popup container. Useful for custom styling, layout adjustments, or z-index control (e.g., z-[200]). Applied to the outer fixed container element.
panelClassName string '' Optional additional CSS classes to apply to the inner popup panel. Useful for controlling the panel dimensions, padding, or other styles (e.g., !max-w-[400px]). Applied alongside variant-specific theme classes and animation classes.
icon string 'close' The name of the icon to display when showModalIcon is true. Uses the design system's icon set (e.g., 'info-circle', 'check-circle', 'warning', 'close'). The icon appears inside the colored circular background above the heading.
type 'success' | 'warning' | 'error' | 'info' 'info' The semantic type of the popup, which determines the color scheme of the modal icon background when showModalIcon is true:
  • 'success': Green background for success messages
  • 'warning': Amber/yellow background for warnings
  • 'error': Red background for error messages
  • 'info': Blue background for informational messages
isFullScreen boolean false When true, the popup panel expands to full screen width and height. Particularly useful for slide variants (slideLeft, slideRight) when you want the panel to cover the entire viewport instead of a fixed width.
