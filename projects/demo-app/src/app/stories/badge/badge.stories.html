# API Reference

Property Type Default Description
Component Inputs
label string - Required. The text content displayed in the badge. This is the primary visible text label for the badge.
variant string 'default' Visual style variant of the badge. Available options: 'default', 'amber', 'blue', 'green', 'violet', 'red', 'orange'. Each variant has a distinct background and text color combination.
size string 'md' Size of the badge. Available options: 'sm', 'md', 'lg'. Controls padding and text size.
icon string undefined Optional icon identifier displayed in the badge. Should match an available icon in your icon library (e.g., 'check', 'close', 'message'). Position is controlled by iconRight.
iconRight boolean false When true, displays the icon on the right side of the label. When false (default), the icon appears on the left side. Only applies when icon is provided.
pill boolean false When true, applies fully rounded corners (pill shape) to the badge. When false, uses slightly rounded corners. Use for a more rounded, pill-like appearance.
withDot boolean false When true, displays a small colored dot before the label. The dot color matches the badge variant. Use to add visual emphasis or indicate status.
href string undefined Optional URL or route path. When provided, the badge renders as an anchor tag (<a>) instead of a span, making it clickable and navigable. Use for badges that should link to other pages or resources.
className string '' Optional additional CSS classes to apply to the badge container. Useful for custom styling or layout adjustments.
Component Outputs (Events)
onClick EventEmitter<Event> - Emitted when the badge is clicked. Only fires when href is not provided (badge renders as a span, not an anchor). Provides the click event object. Use for custom click handling when the badge is not a link.
