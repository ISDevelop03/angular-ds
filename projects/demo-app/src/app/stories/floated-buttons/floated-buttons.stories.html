Floated Buttons

# API Reference

Property Type Default Description
Component Inputs
items FloatedButton[] [] Required. Array of button objects to display in the floated buttons container. Each button can have text, icons, loading states, and click handlers. Buttons are displayed horizontally with vertical dividers between them. The component only renders if items exist and have length. See the FloatedButton Interface section below for details.
className string '' Optional additional CSS classes to apply to the floated buttons container. Useful for custom styling or positioning (e.g., absolute bottom-6 right-0 left-0 mx-auto for floating positioning). The container has default styling with border, shadow, and background.

# FloatedButton Interface

The FloatedButton interface defines the structure for button items passed to the items input property.

Property Type Default Description
content string - Required. The button label text displayed on the button. This is the primary visible text content for the button.
onClick (event: any) => void undefined Optional callback function executed when the button is clicked. Receives the click event as a parameter. Use to handle button clicks and perform actions.
type 'button' | 'submit' | 'reset' undefined Optional HTML button type attribute. Available options: 'button', 'submit', 'reset'. Use 'submit' for form submission buttons. When not provided, defaults to 'button'.
icon string undefined Optional icon identifier displayed on the button. Should match an available icon in your icon library (e.g., 'edit', 'delete', 'save'). The icon position is controlled by iconRight. When isLoading is true, the icon is replaced with a loading spinner.
iconRight boolean undefined When true, displays the icon on the right side of the button text. When false or not provided, displays the icon on the left side. Only applies when icon is provided and isLoading is false.
isLoading boolean undefined When true, displays a loading spinner icon (Loader) with animation instead of the regular icon. The spinner appears on the left side of the button text. Use to indicate that an action is in progress. When true, the regular icon is hidden.
isDelete boolean undefined When true, applies red text color styling to the button (text-red-500). Use to indicate destructive actions like delete operations. When false or not provided, uses the default text color.
disabled boolean undefined When true, disables the button, preventing user interaction. The button appears grayed out with reduced opacity and a not-allowed cursor. Use to prevent actions in certain states.
className string undefined Optional CSS classes to apply to the individual button. Use for custom styling of specific buttons. Applied in addition to the default button classes and the isDelete styling.
