# API Reference

Property Type Default Description
Component Inputs
items ITab[] [] Required. Array of tab items to display. Each item defines a tab with its title, panel content, and optional properties. The tabs are displayed in the order they appear in this array. See the ITab Interface section below for details.
className string '' Optional additional CSS classes to apply to the tabs wrapper container. Useful for custom styling, spacing, or layout adjustments. The component has default styling based on the variant property.
variant 'default' | 'noBackground' | 'withBorder' 'default' Theme variant for styling the tabs component. Options:
  • 'default': Tabs with background container and rounded styling
  • 'noBackground': Tabs without background, with underline indicator
  • 'withBorder': Tabs with border container and underline indicator
Each variant has different visual styling for the tab list and panels.
rightContent TemplateRef<any> undefined Optional template reference for content displayed on the right side of the tab list. This allows you to add custom content (e.g., buttons, icons, actions) next to the tabs. The content is rendered using ngTemplateOutlet and appears in a flex container with the tab list.
selectedIndex number 0 The index of the currently selected tab. This is a zero-based index (0 for the first tab, 1 for the second, etc.). The component automatically updates this value when a tab is clicked, but you can also control it externally to programmatically select a specific tab. The selected tab's panel is displayed, and other panels are hidden.
Component Outputs (Events)
onSelect EventEmitter<any> - Emitted when a tab is selected (clicked). The event is only emitted for non-disabled tabs without href (link tabs don't emit this event). The payload contains: { event: Event, tab: ITab, index: number }. Use to handle tab selection, track user interactions, or perform actions when switching tabs.

# ITab Interface

The ITab interface defines the structure for tab items passed to the items input property.

Property Type Default Description
title string - Required. The title text displayed in the tab button/link. This is the label shown in the tab header (e.g., "Tab 1", "Settings", "Profile").
panel string | TemplateRef<any> - Required. The content to display in the tab panel. Can be either:
  • String: HTML content rendered using innerHTML. Use for simple HTML content.
  • TemplateRef: Angular template reference for complex content. Use for dynamic content, components, or interactive elements. The template is rendered using ngTemplateOutlet.
The panel is only displayed when the tab is selected (based on selectedIndex).
href string undefined Optional URL for link tabs. When provided, the tab is rendered as an anchor tag (<a>) instead of a button, and clicking it navigates to the URL. Link tabs don't trigger the onSelect event and don't change the selectedIndex. Use for external links or navigation that should use browser routing.
link string undefined Optional Angular router link for link tabs. When provided along with href, the tab uses Angular's routerLink directive for navigation. This allows for client-side routing without page reloads. The tab is still rendered as an anchor tag, but uses Angular routing instead of standard navigation.
disabled boolean undefined When true, disables the tab, preventing clicks and showing a disabled visual state (reduced opacity, not-allowed cursor). Disabled tabs don't trigger the onSelect event and can't be selected. Use for tabs that should be temporarily unavailable or require certain conditions to be met.

Additional Notes:

  • Tab Types: The component supports two types of tabs:
    • Button tabs (default): Rendered as buttons that trigger tab selection and emit the onSelect event
    • Link tabs (when href is provided): Rendered as anchor tags for navigation, don't emit events or change selection
  • Panel Content: Tab panels can display content in two ways:
    • String content: HTML strings are rendered using innerHTML, allowing for simple HTML markup
    • Template content: TemplateRef allows for complex Angular templates with components, directives, and bindings
    Only the selected tab's panel is displayed; other panels are hidden using the [hidden] directive.
  • Tab Selection: Tab selection is controlled by:
    • The selectedIndex input property
    • User clicks on tab buttons (updates selectedIndex internally)
    • The onSelect event emission when a tab is clicked
    You can programmatically select a tab by updating the selectedIndex input.
  • Variants: The component provides three visual variants:
    • default: Tabs with a background container (bg-level-3), rounded styling, and selected tabs with red background and text
    • noBackground: Tabs without background, with an underline indicator that appears on selection/hover
    • withBorder: Tabs with a border container (bg-crame-100), with an underline indicator that appears on selection/hover
  • Right Content: The rightContent template allows you to add custom content on the right side of the tab list. This is useful for:
    • Action buttons (e.g., "Add", "Settings", "More")
    • Icons or indicators
    • Custom controls or filters
    The content is displayed in a flex container alongside the tabs.
  • Accessibility: The component includes accessibility features:
    • aria-selected attribute on tab buttons to indicate selection state
    • aria-disabled attribute on disabled tabs
    • disabled attribute on disabled tab buttons
    • Focus management for keyboard navigation
  • Link Tabs: When a tab has an href, it's rendered as a link instead of a button. Link tabs:
    • Don't trigger the onSelect event
    • Don't update the selectedIndex
    • Use standard browser navigation or Angular routing (if link is also provided)
    • Still respect the disabled state
  • Styling: Each variant has specific styling for:
    • Tab buttons/links (base, selected, unselected states)
    • Tab list container (background, padding, spacing)
    • Panel container and panels (padding, margins, styling)
    The styling uses Tailwind CSS classes and can be customized through the theme configuration.
  • Template References: When using TemplateRef for panel content, you typically:
    1. Create an ng-template with a template reference variable (e.g., #tabOne)
    2. Pass the template reference to the panel property in the items array
    3. The component automatically detects and renders the template
  • State Management: The component manages tab selection state internally, but you can control it externally by binding to the selectedIndex input. This allows for:
    • Two-way data binding with [(selectedIndex)]
    • Programmatic tab selection
    • Synchronization with external state management
