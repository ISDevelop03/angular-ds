# API Reference

Property Type Default Description
Component Inputs
items DropdownItem[] [] Required. Array of dropdown menu items to display. Each item represents a clickable option in the dropdown menu. Items can have labels, icons, click handlers, and nested sub-items. See the DropdownItem Interface section below for details.
variant keyof typeof dropdown.variants 'base' Visual style variant of the dropdown trigger button. Available options: 'base', 'select'. The 'base' variant has no additional styling. The 'select' variant applies border and padding styling similar to a select input.
className string '' Optional additional CSS classes to apply to the dropdown container. Useful for custom styling or layout adjustments.
Content Projection
ng-content - - The dropdown trigger button uses content projection. You can project any content (icons, text, or both) into the trigger button. For example: <ds-icon id="more-1"></ds-icon> or <span>Click me</span>. The projected content becomes the clickable trigger for the dropdown menu.

# DropdownItem Interface

The DropdownItem interface defines the structure for menu items passed to the items input property.

Property Type Default Description
label string - Required. The display text for the menu item. Shown in the dropdown menu. This is the primary visible text label for the item.
icon string undefined Optional icon identifier displayed before the label. Should match an available icon in your icon library (e.g., 'edit', 'delete', 'more'). The icon appears on the left side of the label with proper spacing.
onClick (item: any) => void undefined Optional callback function executed when the menu item is clicked. Receives the click event as a parameter. When provided, the dropdown menu automatically closes after the handler executes. Use to handle item selection and perform actions.
items DropdownItem[] undefined Optional array of nested dropdown items. When provided, creates a nested submenu for this item. Use to create hierarchical dropdown menus with multiple levels. The nested items follow the same structure as the parent items.
