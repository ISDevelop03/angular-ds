# API Reference

Property Type Default Description
Component Inputs
percentage number 0 The progress value. When total is provided, this represents the current value (e.g., 30 out of 60). When total is not provided, this represents the percentage directly (0-100). The component automatically calculates the percentage for display: if total exists, it calculates (percentage / total) * 100; otherwise, it uses percentage directly.
total number undefined Optional total value for progress calculation. When provided, the component calculates the percentage as (percentage / total) * 100. The display shows both values: percentage / total (e.g., "30/60"). When not provided, the component uses percentage directly as the percentage value (0-100). Useful for showing progress as a fraction (e.g., 30 items out of 60 total).
variant 'neutral' | 'success' | 'warning' | 'error' 'neutral' Visual style variant for the progress bar. Determines the color of the progress indicator:
  • 'neutral': Gray progress bar (default)
  • 'success': Green progress bar for successful operations or completion
  • 'warning': Yellow progress bar for warnings or caution
  • 'error': Red progress bar for errors or failures
Each variant has distinct color styling for the progress indicator.
type 'inline' | 'block' 'block' Layout display type for the progress component:
  • 'block': Vertical layout with label/value above the progress bar. The label (if provided) and value are displayed in a row above the progress bar.
  • 'inline': Horizontal layout with progress bar and value side-by-side. The progress bar and value are displayed in a flex row with gap spacing.
label string undefined Optional label text displayed above the progress bar (only for type="block"). Shown on the left side of the header row, with the value on the right. When provided, the header uses space-between justification. When not provided, only the value is shown, right-aligned. Use to provide context or description for the progress (e.g., 'With total example', 'Upload Progress').
suffix string '' Optional suffix text appended to the percentage value. Displayed immediately after the percentage number (e.g., '%' for "30%", 'GB' for "5GB"). The suffix appears before the total fraction if total is provided (e.g., "30%/60" or "30% / 60"). Useful for units or symbols.
className string '' Optional additional CSS classes to apply to the progress component wrapper. Useful for custom styling, layout adjustments, or responsive sizing. The component has default styling with full width. When type="inline", flex layout with gap is applied.

Additional Notes:

  • Progress Calculation: The component automatically calculates the visual progress percentage:
    • When total is provided: (percentage / total) * 100
    • When total is not provided: Uses percentage directly (should be 0-100)
    The calculated percentage is used to set the width of the progress indicator bar.
  • Value Display: The displayed value format depends on inputs:
    • Without total: percentage + suffix (e.g., "30%")
    • With total: percentage + suffix + "/" + total (e.g., "30%/60" or "30% / 60")
  • Progress Bar: The progress bar consists of:
    • A background bar (container) with rounded corners and stroke background
    • A progress indicator (filled portion) with variant-specific color, positioned absolutely, and animated with smooth transitions
    The progress indicator width is set dynamically based on calcPercentage.
  • Animations: The progress indicator has smooth transitions for both color and width changes. When the percentage changes, the width animates smoothly over 300ms with ease-in-out timing. This provides a polished user experience when updating progress values.
  • Layout Differences:
    • Block type: Label and value are shown above the progress bar in a horizontal row. The value is right-aligned if no label, or space-between if label exists.
    • Inline type: Progress bar and value are shown side-by-side in a flex row with gap spacing. No label is displayed in inline mode.
  • Accessibility: The progress bar uses semantic HTML and can be enhanced with ARIA attributes for screen readers. Consider adding role="progressbar", aria-valuenow, aria-valuemin, and aria-valuemax attributes for better accessibility.
