# API Reference

Property Type Default Description
Component Inputs
items ShortcutItem[] - Required. Array of shortcut items displayed in a 2-column grid. Each item can be either a regular shortcut (with isShortcut: true) or a placeholder for adding new shortcuts (with isShortcut: false). Regular shortcuts are clickable and can be removed. Placeholders open a dropdown to select new shortcuts. See the ShortcutItem Interface section below for details.
shortcuts DropdownShortcut[] - Required. Array of available shortcuts displayed in the dropdown menu when a placeholder item is clicked. Each shortcut can be selected to add it to the items array. Shortcuts that are already selected (present in items) should have isSelected: true. See the DropdownShortcut Interface section below for details.
className string '' Optional additional CSS classes to apply to the grid container holding the shortcut items. Useful for custom styling, layout adjustments, or responsive sizing (e.g., max-w-[325px]). The component has default styling with a 2-column grid layout and gap spacing.
Component Outputs (Events)
click EventEmitter<any> - Emitted when various actions occur in the shortcuts component. Provides different payloads based on the action:
  • Shortcut click: When a regular shortcut item (with isShortcut: true) is clicked, emits the item with its index: { ...item, idx: number }
  • Navigation: When a non-shortcut item is clicked, emits: { action: 'navigate', item: ShortcutItem, idx: number }
  • Remove: When a shortcut's remove button is clicked, emits: { action: 'remove', item: ShortcutItem, idx: number }
  • Add: When a shortcut is selected from the dropdown, emits: { action: 'add', shortcut: DropdownShortcut, placeholderIdx: number, idx: number }
Use to handle navigation, removal, addition, or other shortcut actions.
removed EventEmitter<{ item: DropdownShortcut; idx: number }> - Emitted when a shortcut is removed from the items array. Provides an object with the removed item and its index: { item: DropdownShortcut, idx: number }. The component automatically removes the item from the items array using splice. Use to track removals, update state, or perform cleanup actions.

# ShortcutItem Interface

The ShortcutItem interface defines the structure for shortcut items passed to the items input property.

Property Type Default Description
icon string - Required. Icon identifier displayed in the shortcut card. Should match an available icon in your icon library (e.g., 'home', 'settings', 'user'). The icon is displayed in a square container (28px Ã— 28px) with green color and background.
title string - Required. The title text displayed in the shortcut card. Shown next to the icon in semibold text. This is the label for the shortcut (e.g., 'Home', 'Settings', 'Profile'). When href is provided and isShortcut is true, the title is rendered as a link.
href string undefined Optional URL or route path. When provided and isShortcut is true, the title is rendered as an anchor tag for navigation. Can be a relative path, absolute URL, or route path depending on your routing setup. The link covers the entire card area via absolute positioning.
isShortcut boolean undefined When true, the item is a regular shortcut that can be clicked and removed. When false or undefined, the item is a placeholder that opens the dropdown menu when clicked to add a new shortcut. Regular shortcuts have different styling (darker background) and show a remove button on hover. Placeholders have lighter background styling.

# DropdownShortcut Interface

The DropdownShortcut interface defines the structure for available shortcuts passed to the shortcuts input property. These shortcuts appear in the dropdown menu when a placeholder is clicked.

Property Type Default Description
title string - Required. The title text displayed in the dropdown menu item. Shown in bold text. This is the label for the available shortcut option (e.g., 'Home', 'Settings', 'Profile').
href string undefined Optional URL or route path. When provided, the shortcut can be used for navigation. Can be a relative path, absolute URL, or route path depending on your routing setup. This href can be passed to the ShortcutItem when the shortcut is added to the items array.
isSelected boolean undefined When true, indicates that this shortcut is already present in the items array. Selected shortcuts are displayed with reduced opacity, disabled cursor, and a minus icon (red) instead of an add icon. They cannot be selected again. Use to prevent duplicate shortcuts.
onClick (data: any) => void - Required. Callback function executed when the shortcut is selected from the dropdown. Receives an object with shortcut and idx properties: { shortcut: DropdownShortcut, idx: number }. The component automatically calls this handler after emitting the click event with action 'add'. Use to handle shortcut selection, update state, or perform additional actions.

Additional Notes:

  • Grid Layout: The shortcuts are displayed in a 2-column grid layout. Each shortcut item is a card with an icon, title, and optional remove button. The grid automatically wraps items to new rows as needed.
  • Shortcut Types: Items can be either:
    • Regular shortcuts (isShortcut: true): Active shortcuts that can be clicked for navigation and removed via the delete button that appears on hover
    • Placeholders (isShortcut: false or undefined): Empty slots that open a dropdown menu when clicked to add new shortcuts
  • Dropdown Menu: When a placeholder is clicked, a dropdown menu appears below the shortcuts grid showing all available shortcuts from the shortcuts array. The dropdown:
    • Shows each shortcut with its title and an add/minus icon
    • Disables already-selected shortcuts (with reduced opacity)
    • Closes when a shortcut is selected or when clicking outside
  • Remove Functionality: Regular shortcuts have a remove button (minus icon) that appears on hover in the top-right corner. When clicked, it:
    • Emits the click event with action 'remove'
    • Emits the removed event
    • Removes the item from the items array
  • Click Outside: The component listens for document clicks and automatically closes the dropdown menu when clicking outside the component. This provides a better user experience.
  • Hover Effects: Regular shortcuts show a remove button on hover with smooth transitions. The remove button slides down and fades in when hovering over a shortcut card.
  • Styling Differences:
    • Regular shortcuts: Darker background (bg-level-5), green icon container with darker background
    • Placeholders: Lighter background (bg-crame-100), green icon container with lighter background
  • Event Propagation: All click events use stopPropagation() to prevent event bubbling, ensuring that clicks on shortcuts don't trigger parent element handlers.
