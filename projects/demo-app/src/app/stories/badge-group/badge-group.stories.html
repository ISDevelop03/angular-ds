Badge Group

# API Reference

Property Type Default Description
Component Inputs
tags IBadge[] [] Required. Array of badge items to display in the group. The component automatically handles overflow by showing visible badges and a "+N" button for overflow items. When the "+N" button is clicked, a dropdown appears with the hidden badges. See the IBadge Interface section below for details.
className string '' Optional additional CSS classes to apply to the badge group container. Useful for custom styling or layout adjustments (e.g., setting borders: border border-neutral-100 max-w-fit rounded p-3).
moreClassName string '' Optional CSS classes to apply to the "+N" overflow button. Use for custom styling of the button that appears when badges overflow the container width.
moreIcon string 'more' Icon identifier for the overflow button. Should match an available icon in your icon library. Defaults to 'more'. The overflow button displays this icon alongside the count of hidden badges.
moreIconSize string '20px' Size of the overflow button icon. Accepts any valid CSS size value (e.g., '20px', '1rem', '24px'). Use to adjust the icon size to match your design requirements.

# IBadge Interface

The IBadge interface defines the structure for badge items passed to the tags input property. This interface matches the badge component's properties.

Property Type Default Description
label string - Required. The text content displayed in the badge. This is the primary visible text label for the badge.
variant string - Required. Visual style variant of the badge. Available options: 'default', 'amber', 'blue', 'green', 'violet', 'red', 'orange'. Each variant has a distinct background and text color combination.
size string - Required. Size of the badge. Available options: 'sm', 'md', 'lg'. Controls padding and text size. Defaults to 'md' if not specified.
pill boolean - Required. When true, applies fully rounded corners (pill shape) to the badge. When false, uses slightly rounded corners.
outline boolean - Required. When true, applies outline styling to the badge (border with transparent background). When false, uses filled background styling.
withDot boolean - Required. When true, displays a small colored dot before the label. The dot color matches the badge variant. Use to add visual emphasis or indicate status.
iconRight boolean - Required. When true, displays the icon on the right side of the label. When false, the icon appears on the left side. Only applies when icon is provided.
className string - Required. Additional CSS classes to apply to the individual badge. Use for custom styling of specific badges.
icon string undefined Optional icon identifier displayed in the badge. Should match an available icon in your icon library (e.g., 'check', 'close', 'message'). Position is controlled by iconRight.
href string undefined Optional URL or route path. When provided, the badge renders as an anchor tag, making it clickable and navigable. Use for badges that should link to other pages or resources.
